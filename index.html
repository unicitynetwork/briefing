Developer Platform MVP: Marketplace, Dock & UI Overhaul
PR #335 introduces the developer platform MVP, delivering three major capabilities in a single merge: a marketplace for discovering and installing developer tools, a desktop dock for quick-access navigation, and a maintenance mode for controlled downtime workflows. The work also formalises adoption of sphere-ui as the canonical component library across the platform, replacing ad-hoc styling with consistent, reusable primitives. This is a foundational milestone that establishes the surface area on which third-party and internal developer tooling will be built and distributed.
unicity-sphere/sphere
Sphere SDK Bump to 0.7.2
PR #337 is a dependency bump, updating @unicitylabs/sphere-sdk from its previous version to 0.7.2. Keeping the SDK current ensures the sphere frontend benefits from the latest API contracts, type definitions, and any upstream bug fixes or performance improvements shipped in the 0.7.x line. Though routine in scope, this chore reduces drift between the client and the SDK and prevents compatibility issues from accumulating.
unicity-sphere/sphere
S3 Media Upload Operator Reference Docs
PR #27 adds operator-facing reference documentation covering S3 media upload integration in the sphere-api layer. The docs detail configuration expectations, credential handling, and the upload flow that operators must follow to enable media storage for their deployments. Clear operator reference material lowers the barrier to self-service deployment and reduces support burden as the platform scales to external operators.
unicity-sphere/sphere-api
Migrate Twitter API Calls to x.com Domain
PR #26 updates all Twitter API call targets in sphere-api from the legacy twitter.com endpoints to the current x.com domain. This is a necessary correctness fix following Twitter's domain migration, ensuring API requests are not silently redirected or rejected by DNS and TLS changes on the platform side. Without this change, any integration relying on Twitter data — such as social authentication or feed embedding — risked latency spikes or outright failures.
