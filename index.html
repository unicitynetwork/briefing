Sphere15 PRs merged
S3 image upload pipeline: presign, confirm, delete
PR #25 in sphere-api introduced the full S3 image upload backend: presigned URL generation, upload confirmation, deletion endpoints, a background worker, and replacement hooks to swap old assets cleanly. PRs #7 (sphere-backoffice) and #6 (sphere-dev-portal) wired this backend into every admin and developer-portal form, giving operators a consistent upload experience across both surfaces. Together these three PRs close the gap between raw file storage and the admin tooling, removing any previous workaround of embedding base64 or external URLs directly in form payloads.
sphere-api, sphere-backoffice, sphere-dev-portal
MediaUploader, ProjectLogo, and marketplace preview components
Six consecutive sphere-ui PRs (#4 through #10) built out the media and marketplace UI layer. PR #4 ported marketplace preview components 1:1 from the Sphere wallet, PR #5 added deferred upload, tagging, and general UI polish to MediaUploader, and PR #6 fixed banner overlay alignment, logo padding, and a nested-button accessibility violation. PR #8 extracted a dedicated <ProjectLogo> component so the same tile visual renders consistently everywhere, PR #9 bundled MediaUploader clear behaviour and ProjectPagePreview logo polish into the v0.1.22 release, and PR #10 corrected peer dependencies that were mistakenly listed as external, fixing downstream install warnings. A stray manual version bump introduced in a prior commit was reverted in PR #7 to restore the automated publish workflow as the single source of version truth.
sphere-ui
Agentic-chatbot stability: memory, disk, and Node 22 upgrade
PR #10 in agentic-chatbot bounded the posting loop that was causing unbounded heap growth (tracked as issue #7), and PR #9 capped container log file size to prevent unbounded disk consumption on long-running deployments (issue #8). PR #11 upgraded the unicity-l3/viktor/kbbot Docker base image to node:22-alpine, a hard requirement because pnpm 11 dropped support for older Node versions. The sphere-infra submodule was bumped in PR #5 to commit 09ee153, pulling in all three fixes and making the corrected chatbot the live version in the integrated stack.
sphere-infra, agentic-chatbot
Self-hosted Grafana metrics stack replaces Grafana Cloud
PR #4 in sphere-infra delivered a fully self-hosted Grafana and metrics stack, replacing the previous dependency on Grafana Cloud. This eliminates recurring SaaS costs and keeps all telemetry data within the project's own infrastructure boundary. The change gives the team direct control over retention policies, dashboards, and alerting rules without being subject to Grafana Cloud tier limits.
