Astrid5 PRs merged
MCP Server: Claude-Facing Capsule Tool Exposure
PR #880 introduces `astrid mcp serve`, a Model Context Protocol server that exposes capsule tools and wires consent flows directly to Claude. This gives AI agents a structured, audited surface to invoke Astrid capabilities without bypassing capability gating. The work builds on the persistent-stdin plumbing landed in PR #871, which added acceptance tests for persistent stdin and un-stubbed the `write-stdin`/`close-stdin` WIT bindings. PR #14 in the WIT repo follows up with documentation corrections to reflect those bindings as fully implemented rather than stubbed.
unicity-astrid
Persistent Process Grants and Spawn Gating
PR #873 adds an `allow_persistent` operator sub-grant that gates the `spawn-persistent` capability, giving capsule authors fine-grained control over which sub-processes may outlive their parent invocation. This closes a capability-escalation gap where a capsule could previously spawn persistent processes without an explicit operator endorsement. The sub-grant model follows the existing grant hierarchy and is exercised by the acceptance tests introduced in PR #871.
unicity-astrid
WIT Submodule Hygiene Fix
PR #876 re-points the WIT submodule to the canonical `main` branch at commit `83ebc6c`, correcting a drift that left the workspace referencing a detached or non-canonical revision. This is a low-risk housekeeping change but is a prerequisite for downstream consumers and CI pipelines that resolve WIT interfaces by submodule SHA.
unicity-astrid, wit

Sphere22 PRs merged
UXF Swap Pipeline: Soak Testing and CLI Controls
PR #439 introduces a full swap-roundtrip soak test suite and a demo playbook tracking issue #437, validating the UXF swap flow under sustained load. PR #449 fixes a poison-pill scan in soak section 10 that was incorrectly aborting on otherwise clean runs. PR #436 corrects CID encoding by representing UXF child and predecessor references as CBOR Tag 42 CID-links rather than raw bytes, which is required for correct IPLD traversal. On the CLI side, PR #44 in sphere-cli adds `swap reject`, `swap cancel`, and `swap wait` commands, giving operators direct control over in-flight swap lifecycle from the terminal.
sphere-sdk, sphere-cli
Transport and Relay Stability Fixes
PR #443 gates the mux relay subscription until module handlers have fully registered, eliminating a race where relay messages arrived before handlers were ready to process them (issue #442). PR #446 changes `getSwapStatus` to lazy-load terminal swap records from storage rather than holding them in memory, fixing a regression tracked in #445 where swaps could appear missing after a restart. PR #428 gates Nostr subscriptions on handler readiness (issue #423) using the same pattern, and PR #429 makes `AggregatorPinger` resilient to transient timing flakes (issue #424).
sphere-sdk
Profile, Identity, and Payments Hardening
PR #385 removes identity and seed material from OrbitDB entirely, ensuring sensitive key material is never persisted in the distributed log. PR #422 pins OpLog blocks at `depth=0` and emits a one-shot Helia config snapshot to fix profile pointer reliability (issue #419). PR #338 adds a PID-liveness probe that clears stale CLI file locks on startup, resolving the deadlock scenario described in issue #336. On the payments side, PR #438 ships v2 self-mint and top-up flows, and PR #431 stabilises the transient-classification assertion in the negative-LRU test to eliminate flakiness.
sphere-sdk
Network Config, Aggregator API Key, and Test Isolation
PR #441 adds a `testnet2` network preset, aligning the SDK with the newly deployed sharded testnet infrastructure. PR #440 removes the hardcoded aggregator API key default and requires an explicitly injected value, preventing accidental use of a shared credential in production builds. PRs #432 and #433 introduce per-test `TEST_DIR` isolation for eight previously flake-prone integration test files, addressing intermittent failures caused by shared filesystem state across parallel test runs. PR #192 promotes the `errMessage` helper to a shared utility and fixes lossy stringification in `NametagMinter`.
sphere-sdk

Testnet2 Sharded BFT Deployment and Infrastructure
PR #2 in bft-terraform provisions a self-contained sharded testnet v2 environment in `eu-central-1`, and bft-ansible PR #2 delivers the full Ansible deploy for an 8-shard × 2 HA BFT network with self-hosted observability. Subsequent bft-ansible PRs (#3–#7) harden the deployment: PR #3 trims monitoring cardinality from ~50.6k to ~29k series by dropping MongoDB collectors and adding Alloy drop-rules; PR #4 wires 17 Grafana-managed alerting rules to Discord; PR #5 fixes a dead `ShardNoLeader` alert and adds secret/idempotency/fail-loud hardening. PRs #6 and #7 bump the gateway binary to commit `9de0820` (carrying a CORS `X-State-ID` fix) and raise the gateway `--idle-timeout` to 75000 ms to exceed the 60-second ALB idle timeout and avoid 502 errors on connection reuse.
bft-terraform, bft-ansible
Aggregator CORS, Hotfixes, and Subscription Backports
PR #38 adds `X-State-ID` to the CORS preflight allowed-headers list and makes the full allowed-headers set env-configurable, unblocking browser clients that set this header. PR #39 backports v1-legacy hotfixes — including the CORS fix — to `main` for testnet v2 compatibility, ensuring the subscription service on the new network benefits from fixes originally developed against the production v1 path.
aggregator-subscription
Observability: VictoriaMetrics Migration and ML Inference Scaling
PR #61 migrates metrics collection for both staging and production to a self-hosted VictoriaMetrics instance, removing a dependency on an externally managed metrics backend and reducing per-series cost. PR #60 raises the PromptShield CPU cap from 4 to 8 cores on both prod and staging to accommodate ML inference workloads without throttling. PR #56 from @lploom introduces a dedicated ML inference pool, separating inference compute from general service capacity and allowing independent scaling.
