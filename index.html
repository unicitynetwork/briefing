Sphere20 PRs merged
Fail-loud network config replaces silent defaults
PRs #467–#471 in sphere-sdk and #341–#342 in sphere deliver a three-phase overhaul of network configuration handling. Phase 1 (#467) added regression guards for the network-drop bug class; Phase 2 (#469) removed all silent testnet/mainnet fallbacks so missing network config throws immediately; Phase 3 (#471) made NetworkConfig a required parameter and added a startup consistency assertion. sphere #341 threads network through Sphere.import and importFromLegacyFile, and #342 bumps the SDK to 0.8.0-dev.5. Together these changes eliminate an entire class of silent misconfiguration bugs where the wrong network could be used without any error.
sphere-sdk, sphere
Registry and icon loading fixed across networks
PR #466 in sphere-sdk forwards the network parameter through the init→create/load path and introduces a per-network cache, fixing missing testnet2 icons and mint data. sphere #340 ensures Sphere.init passes network down so the registry correctly loads testnet2 assets, resolving an icons regression. sphere #338 separately fixes token symbol resolution for fresh wallets, eliminating the need to re-login before symbols appear. PR #470 in sphere-sdk fixes the integration test suite to use a shared TEST_NETWORK constant so tests remain consistent with the new fail-loud behaviour.
sphere-sdk, sphere
Payments, swaps, and profile reliability hardened
PR #453 splits the payments flow into separate local-commit and remote-publish phases, fixing a race condition tracked in sphere-sdk#444. #461 ensures terminal swaps are included in resolveSwapId and getSwaps results and adds write-side error codes (follow-up to #447). #459 adds fail-fast behaviour when transportPubkey is missing in the swap path (#457). #452 bounds the pre-shutdown publish-retry spin in the profile module to prevent indefinite loops (#450), and #463 addresses HIGH-severity findings surfaced in the post-merge review of #453. #451 corrects v2 send gating to check recipient chainPubkey rather than DirectAddress.
sphere-sdk
Transport, faucet docs, escrow fallback, and group chat
PR #465 fixes an await gap in the mux dispatch path that could cause dropped messages under concurrent transport use (sphere-sdk#464). #460 adds an assertion to the test suite confirming the mux subscription gate stays closed while modules are loading (#448). #462 documents the investigation into single-coin faucet flakiness in the UXF layer (#455), and #458 documents the escrow DIRECT address fallback used during nametag resolution (#456). sphere #339 disables Group Chat entirely by hiding the desktop icon and blocking access, removing a feature that was not ready for users.
sphere-sdk, sphere

Unicity Network5 PRs merged
End-to-end h2c support for testnet2 gateway layer
PR #41 in aggregator-subscription extends the shard connectivity probe to operate over h2c when --upstream-h2c is set, fixing startup failures when upstreams speak HTTP/2 cleartext. PR #42 extracts a shared AggregatorBlockHeightProbe and makes the /health endpoint h2c-aware so liveness checks work correctly in h2c deployments. On the infrastructure side, bft-ansible #8 bumps the testnet2 gateway image to commit f580750, which is a prerequisite for the validator h2c fix, and #9 completes the cutover by enabling the gateway→HAProxy h2c path on testnet2. Together these four PRs close the full h2c upgrade path from the validator through to the health probe.
aggregator-subscription, bft-ansible
Java SDK gains networkId with transaction-level verification
PR #69 resolves issue #68 by adding a networkId field to the state-transition SDK for Java and wiring in verification logic so transactions are checked against the expected network at parse time. Mint transaction parameters are also reordered as part of the same change, likely to align the Java API with the canonical parameter ordering in other SDK implementations. This prevents cross-network transaction replay and makes the Java SDK consistent with network-awareness already present elsewhere in the Unicity stack.
