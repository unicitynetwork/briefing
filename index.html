CBOR Tag Support Across Core and Base Libraries
Two PRs from @MastaP land coordinated CBOR tag support across the BFT stack: #19 in bft-core introduces CBOR tags at the foundational encoding layer, while #4 in bft-go-base refactors the Go base library to align with the new tagging scheme. CBOR tags allow typed, self-describing binary encoding, which is critical for unambiguous serialization of consensus messages and data structures across heterogeneous nodes. The refactoring in bft-go-base (#4) suggests existing encoding logic was updated rather than extended, likely reducing technical debt and ensuring consistency with the core implementation. Together these changes harden interoperability between components that must agree on wire formats during Byzantine fault-tolerant consensus rounds.
