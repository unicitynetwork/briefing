Tombstone Lamport clocks and DoS bounds defense
PR #172 delivers two P1 hardening items: Lamport-clock tombstone support for conflict-free deletions and denial-of-service bounds-checking on inbound messages. These changes close issue #166 and are part of a sequenced P1 hardening series. Together they strengthen the distributed consistency model and prevent resource exhaustion under adversarial load, making the SDK meaningfully more robust for production deployments.
sphere-sdk
Payment dispatcher, receiver restart, and nametag consistency
PRs #147, #146, #140, and #127 collectively overhaul the payments and identity layer. #147 introduces split-aware sender bookkeeping in the UXF dispatcher so multi-part payments are tracked correctly across splits. #146 makes receiver finalization machinery survive process restarts by persisting intermediate state, closing issue #144. #140 preserves nametags when sync payloads omit the _nametags field (#136), and #127 drains stale pending V5 finalizations while enforcing nametag-to-Nostr consistency, eliminating a class of silent data loss bugs.
sphere-sdk
Swap e2e migration to sphere-cli and test infrastructure overhaul
PRs #164, #162, #161, #160, #15, #14, and #13 complete the extraction of swap end-to-end testing from sphere-sdk into sphere-cli. #162 removes the in-tree swap shell scripts now that migration is done, while #164 fixes continuous-swap e2e to skip missing escrows rather than surfacing ENOENT errors (#157). #161 repairs the pointer-N* probe and a dueDate=null EXPIRED bug (#156), and #160 introduces production-shaped fixtures for transfer integration tests (#149). On the sphere-cli side, #13 adds comprehensive post-extraction CLI integration coverage, #14 layers in Docker-based swap e2e alongside crypto/util, init-nametag, group, and market tests, and #15 bumps the default escrow image to the published v0.2 tag, ensuring all tests run against a realistic artefact.
sphere-sdk, sphere-cli
OrbitDB/Helia teardown budget and profile-sync test fixes
PRs #141 and #139 address reliability issues in the profile subsystem. #141 introduces per-step time budgets for OrbitDB and Helia teardown, preventing indefinite hangs during shutdown and closing issue #137. #139 corrects the token.genesis assertion in the profile-sync e2e suite (#138), which was masking real regressions.
