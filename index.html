Maintenance Mode Across Full Sphere Stack
Three PRs from @igmahl (#1 in sphere-quest-frontend, #1 in sphere-backoffice, and #2 in sphere-api) introduce maintenance mode support across the entire Sphere stack in a coordinated, full-stack rollout. The API layer gains the backend logic to toggle and serve maintenance state, while the backoffice receives the administrative controls to manage that state. The frontend quest interface is updated to detect and display the maintenance mode experience to end users. This unified approach ensures consistent behaviour across all layers when maintenance mode is activated, reducing the risk of partial states or mismatched UI signals.

CBOR Tag Encoding for Token Data Elements
PR #7 by @ahtotruu introduces CBOR tags for token data elements in the unicity-ids repository, formalising the binary serialisation schema for identity tokens. Assigning explicit CBOR tags ensures that each data element carries unambiguous type information, improving interoperability with external CBOR-aware tooling and validators. This change strengthens the canonical encoding of token structures, which is critical for deterministic hashing and cross-platform consistency within the Unicity identity layer.
unicity-ids
Sharding Bug Fix in BFT Consensus Core
PR #21 by @lploom addresses a sharding defect in bft-core, targeting correctness in how the Byzantine Fault Tolerant consensus engine partitions or routes data across shards. Sharding bugs at the consensus layer can lead to split-brain scenarios, missed votes, or inconsistent ledger state across nodes, making this a high-priority fix. The change stabilises shard assignment or message routing logic, ensuring that consensus rounds proceed correctly under a sharded topology. This unblocks reliable horizontal scalability for the Unicity network infrastructure.
bft-core
