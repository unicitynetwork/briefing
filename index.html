Sphere23 PRs merged
Intent payload limits and validation hardening
PR #103 introduces an env-tunable 64 KiB per-intent payload cap and a per-owner lifetime byte budget in wallet-api, closing an unbounded ingestion risk. PR #671 in sphere-sdk makes deterministic putIntent VALIDATION rejections terminal and isolates resync replay so transient and permanent failures are no longer conflated. PR #427 bumps sphere-sdk to 0.11.10 in sphere to pull in this intent-validation hardening. Together these changes tighten the contract around intent submission and prevent malformed or oversized intents from propagating through the pipeline.
wallet-api, sphere-sdk, sphere
Payment sync UX and wallet reliability fixes
PR #669 introduces a SEND_SYNC_PENDING tag for post-commit mirror-sync failures in sphere-sdk, giving users a reassuring pending-success state rather than an opaque error. PR #426 bumps sphere to sdk 0.11.9 and surfaces SEND_SYNC_PENDING as a pending-success in the UI. On the reliability side, PR #668 adds retry logic on POST /v1/inventory/apply for transient network failures, safe because the endpoint is idempotent by transferId, documented formally in wallet-api PR #101. PR #666 fixes auth-challenge validation to use server timestamps instead of the device clock, eliminating spurious clock-skew failures, and PR #667 treats a lineage CONFLICT during token-storage save as a benign no-op, both shipped to sphere via PR #421.
sphere-sdk, sphere, wallet-api
Marketplace quest tracks and backoffice admin UX
PR #424 adds quest-track UI and project-page polish to the main sphere frontend, backed by a new quest-tracks endpoint in sphere-api PR #51. Backoffice received significant admin improvements in PR #22, including server-side search, rejection history, reason presets, and quest cap clarity; the companion API changes in PR #50 hide exhausted quests and enable server-side admin search. PR #24 in sphere-dev-portal clarifies that Max Completions is a global cap rather than a per-user limit, reducing operator confusion. All frontends were simultaneously brought up to sphere-sdk 0.11.8 and sphere-ui 0.1.30 via PRs #25, #23, and #12.
sphere, sphere-api, sphere-backoffice, sphere-dev-portal, sphere-quest-frontend
UI polish, Sentry noise reduction, and test reliability
PR #17 in sphere-ui refines logo tile rendering by removing unintended cropping and gloss overlays on real logos and switching to neutral chips, improving visual consistency across the marketplace. PR #428 drops unhandled EIP-1193 provider rejections from injected wallet extensions out of Sentry, eliminating noise generated by third-party browser wallets that is not actionable by the team. PR #429 fixes a flaky pollOrder listener-leak test that suffered a real-timer race condition on loaded CI runners, improving pipeline reliability. PR #99 in wallet-api fixes a validation edge case where §8.2 step 6 incorrectly required a full PaymentAssetCollection and now correctly falls back to the bare variant.
sphere-ui, sphere
