Astrid63 PRs merged
Native OpenAI Responses API capsule ships
PR #1 in capsule-openai introduces a native OpenAI provider built against the Responses API, complete with a model registry and full repo setup, followed by CI in PR #2. On the compatibility side, capsule-openai-compat PR #11 adds IPC-based provider describe and environment-variable generation params, while PR #12 fixes context_window exposure in llm_describe. Together these give Astrid a first-class, standards-aligned path to OpenAI models without relying on the legacy compat shim alone.
capsule-openai, capsule-openai-compat
Astrid 0.5.0 → 0.5.1: CLI, daemon, self-update
PR #615 landed self-update, batch install, TUI fixes and a broad CLI refactor; PR #613 made cargo install astrid pull in both the CLI and daemon binaries in one step, extended by PR #618 to also bundle astrid-build. PR #619 fixed a long-standing bug preventing multiple capsules from exporting the same interface, and PR #606 corrected the HostResult wire format for host-to-guest returns alongside headless TUI support, with the matching SDK decoder fix in sdk-rust PR #25. PR #610 removed the now-redundant [[llm_provider]] block from the kernel manifest, and capsule-registry PR #8 introduced IPC-based LLM provider discovery via hooks::trigger, decentralising provider registration. The 0.5.0 milestone was cut in astrid PR #616 and sdk-rust PR #26 released 0.5.3, with astrid PR #622 following up as 0.5.1 to patch the astrid-daemon crates.io dependency version (PR #617).
astrid, sdk-rust, capsule-react, capsule-registry, capsule-memory, capsule-identity, capsule-session, capsule-router, capsule-agents
Context compaction, identity persistence, streaming fixes
capsule-react PR #11 wires context compaction through the context engine, keeping long-running conversations within model limits automatically. capsule-identity PR #15 adds a save_identity tool so the LLM can persist identity data to disk after onboarding without manual intervention. capsule-react PR #13 fixes a streaming timeout that was incorrectly resetting on every token rather than on silence, preventing spurious disconnects. capsule-memory PR #11 enriches error diagnostics by including the response_topic payload when a topic error occurs, and capsule-identity PR #13 auto-detects an existing spark.toml on the first build request instead of requiring explicit configuration.
capsule-react, capsule-identity, capsule-session, capsule-memory, capsule-context-engine
SDK 0.5.3 rollout, reproducible builds, homebrew tap
All 17 capsule repositories and astralis received a coordinated bump to astrid-sdk 0.5.3 with reproducible build flags enabled, ensuring deterministic artifact hashes across the ecosystem. A new homebrew-tap repository was bootstrapped with PRs #1–#4: the astrid-build formula was added (#1), documented in the README (#2), automated release-dispatch updates were configured (#3), and the formula was bumped to 0.5.1 (#4). Author metadata was corrected across all 17 capsule repos and astralis in a sweep of chore PRs. astralis PR #5 also fixed the init flow to prompt for temperature and max_output_tokens, improving first-run configuration completeness.
