Sphere18 PRs merged
Profile resilience: OpLog reset and corrupt-block tolerance
PRs #301 and #305 harden the profile lifecycle against two distinct failure modes. #301 makes the Sphere-bound factory tolerate corrupt-block bundle refresh errors during lifecycle init rather than crashing. #305 introduces automatic OpLog reset when a missing head block is detected, and permanently stamps a Recovered marker so the condition is traceable. The corresponding vendor bumps in sphere PRs #313 and #316 bring these fixes into the main application.
sphere-sdk, sphere
Migration reliability: flush fix and parallel CAR block pinning
PR #303 addresses an unbound migration flush that could leave profile state inconsistent, and parallelizes CAR block pinning to reduce migration time under load. It also fixes the storage connection ordering in the Sphere-bound factory (PR #300), ensuring storage is connected before tokenStorage.initialize is called to prevent initialization race conditions. Vendor bumps sphere #312 and #315 land both fixes. Migration errors are now surfaced properly in the UXF layer via sphere #310, replacing the previously collapsed Array(2) output that obscured root causes.
sphere-sdk, sphere
Phase 3 walkback skips EXISTS-BUT-UNFETCHABLE versions
PR #302 introduces logic so that during Phase 3 pointer walkback, versions marked EXISTS-BUT-UNFETCHABLE are skipped by default rather than causing a hard failure. This prevents stalled profile recovery when parts of the history are locally unavailable. The fix is vendored into sphere via PR #314. This improves robustness for nodes with incomplete block stores without requiring operator intervention.
sphere-sdk, sphere
UXF wallet-storage controls and SMT path layer separation
PR #294 introduces Sphere-bound factories that keep the privateKey strictly inside the SDK boundary, eliminating private key exposure at the UXF layer; this is vendored via sphere #309 which also drops the legacy privateKey synthesis path. PR #296 embeds InclusionProof.SmtPath as opaque STS-canonical CBOR, achieving full layer separation between the UXF and SDK representations (issue #295), vendored in sphere #311. On the UI side, sphere #308 adds wallet-storage mode status display plus switch, migrate, and merge controls in the banner. sphere-ui PRs #2 and #3 deliver media upload components at v0.1.16 and fix a CI publish regression caused by a manual version bump.
sphere-sdk, sphere, sphere-ui
