Visit-Link Quest Type for LINK Platform
PR #29 introduces a new 'visit link' quest type targeting the LINK platform within the sphere-quest module. This adds an optional configuration layer allowing quest designers to attach external URLs as completable objectives. The change expands the quest engine's supported action vocabulary beyond on-chain interactions into web-based engagement. It lays groundwork for cross-platform campaign mechanics driven by off-chain link visits.

Unicity Network17 PRs merged
ssl-manager: Reusable Startup Library and Multi-Domain Support
PRs #1 through #5 establish the ssl-manager repository as a standalone, reusable SSL lifecycle component. PR #1 introduces run-lib.sh, a generic startup library abstracting certificate acquisition and daemon readiness logic. PR #5 adds SSL_DOMAIN_ALIASES support so a single deployment can serve multiple domains under one certificate workflow. PRs #2 and #4 harden the shell scripts by fixing set -e interactions with the argument parser that caused premature exits on non-zero returns. PR #3 delivers a comprehensive integration guide and CLAUDE.md so downstream repos can adopt ssl-manager with minimal friction.
ssl-manager
Fulcrum-Alpha Docker: Health Checks, DB Recovery, and SSL Integration
PRs #7 through #16 overhaul the Fulcrum-Alpha Docker runtime with production-grade reliability features. PRs #8, #11, and #12 introduce a post-start health check that validates live Electrum protocol responses and renders results with color-coded output for fast operator triage. PRs #9 and #10 add smart DB corruption detection with automatic recovery, followed by steelman hardening of edge cases in the recovery path. PR #13 refactors run-fulcrum.sh to delegate startup logic to ssl-manager/run-lib.sh, eliminating duplication across services. PRs #14 and #16 fix the set -e argument parser bug and an SSL readiness check hang, while PR #15 updates CLAUDE.md to reflect completed specifications.
Fulcrum-Alpha
ipfs-storage: HAProxy Integration, SSL, and GC Lifecycle
PRs #2 and #3 bootstrap operational infrastructure for the ipfs-storage repository. PR #2 wires in ssl-manager integration and HAProxy support, enabling TLS termination and reverse-proxy routing in front of IPFS nodes, alongside targeted IPFS traffic optimisations. PR #3 adds a garbage collection script, manages alias proxy lifecycle to cleanly handle IPFS pin and unpin cycles, and updates run-lib.sh to reflect the new operational model. Together these PRs bring ipfs-storage to the same operational baseline as Fulcrum-Alpha, sharing the common ssl-manager/run-lib.sh foundation.
ipfs-storage
