Permission-Based RBAC Access Control
PR #34 introduced a permission-based role-based access control (RBAC) system to sphere-quest, authored by @KruGoL. This adds fine-grained permission checks on top of existing role assignments, allowing the backend to enforce access policies at the capability level rather than relying solely on broad role membership. The change lays a critical security foundation for multi-tenant quest management, ensuring users can only perform actions explicitly permitted by their assigned roles.
sphere-quest
GitHub Quests Integration
PRs #33 and #32 together delivered GitHub Quests functionality, contributed by @igmahl and @KruGoL respectively. These changes wire sphere-quest into GitHub's API or webhook surface, enabling quests to be defined, tracked, or triggered based on GitHub activity such as commits, pull requests, or repository events. Having two contributors land complementary PRs under the same feature branch suggests a parallel workstream with coordinated integration, accelerating delivery of this external-platform quest capability.
sphere-quest
Wallet Bootstrap from Mnemonic Environment
PR #31 by @KruGoL restored the ability to initialise a wallet directly from a mnemonic phrase supplied via environment variable, combined with an auto-bootstrap sequence. This is significant for deployment automation, as nodes or services can now self-initialise their wallet state without manual intervention at startup. The auto-bootstrap mechanism reduces operator overhead and makes containerised or CI-driven deployments more reliable and reproducible.
