Pubkey-first identity display, DIRECT address hidden
PR #431 removes the raw DIRECT:// address from all UI surfaces and replaces it with a pubkey-first identity display model. This simplifies what users see by surfacing the cryptographic identity rather than the internal transport address, which is an implementation detail irrelevant to end users. The change improves both clarity and security posture by not exposing internal addressing scheme strings in the interface.

Separately, PR #105 in wallet-api introduces a comprehensive observability layer covering size histograms, rejection counters, a config gauge, and high-water gauges across key wallet metrics, backed by nine new alerts. Together these two PRs move the observability stack from reactive noise reduction toward proactive, structured monitoring of wallet behaviour under load.

Unicity Network1 PRs merged
SMT switches to big-endian bit ordering
PR #170 changes the Sparse Merkle Tree implementation in the Go aggregator to use big-endian bit ordering instead of the previous little-endian scheme. Bit ordering in an SMT determines the traversal path for every key, meaning this change affects how all existing and future tree nodes are addressed and proved. Aligning to big-endian ordering is typically necessary for interoperability with other SMT implementations or for consistency with network-level serialisation conventions. This is a low-level but foundational change whose correctness is critical, as any mismatch in bit ordering would produce invalid proofs that silently fail to verify against a counterpart implementation.
