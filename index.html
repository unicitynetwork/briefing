Observability: Prometheus Metrics Endpoint Live
PR #32 by @MastaP introduces a dedicated /metrics HTTP endpoint exposing Prometheus-compatible telemetry for the aggregator-subscription service. This gives operators real-time visibility into aggregator internals without custom instrumentation. Scraping /metrics with standard Prometheus tooling enables alerting, dashboards, and SLA tracking from day one. The addition significantly matures the operational readiness of the aggregator layer.
unicitynetwork/aggregator-subscription
BFT Sharding Routing for Aggregator Subscription
PR #27 by @jait91 implements Byzantine Fault Tolerant sharding routing within the aggregator-subscription component. The change introduces logic to distribute and route subscription traffic across shards in a manner resilient to faulty or malicious nodes. This is a foundational scalability and security improvement, ensuring the aggregator layer can grow horizontally without sacrificing consistency guarantees. BFT routing reduces the risk of single-shard failures cascading across the network.
unicitynetwork/aggregator-subscription
ML-Based Detection in SIF
PR #1 by @ristik introduces machine-learning-based detection capabilities to the SIF (State Integrity Framework) module. This is the first PR in the sif repository, marking the initial build-out of an intelligent anomaly or threat detection layer. ML-driven detection allows the system to identify irregular state transitions or suspicious patterns that rule-based checks would miss. This lays groundwork for adaptive, data-driven security enforcement within the Unicity network.
unicitynetwork/sif
Predicate as Recipient, Mint Reason Refactor
PR #112 by @martti007, resolving issue #110, refactors how recipients and mint metadata are handled in the state-transition-sdk-js. The change replaces raw address-based recipients with Predicate objects, enabling richer, programmable conditions on token receipt. The mint reason field has been moved from the data payload into the mint transaction itself, improving semantic clarity and protocol alignment. These changes tighten the SDK's type model and make mint transactions more expressive and auditable.
unicitynetwork/state-transition-sdk-js
