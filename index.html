Astrid3 PRs merged
Principal-scoped security enforcement across kernel and capsule layers
PRs #667, #669, and #671 together complete a multi-layer principal security model in Astrid, advancing the architecture from per-invocation quota enforcement up through full capability and group gating on the management API. PR #667 introduced per-invocation quota enforcement driven by PrincipalProfile, ensuring each capsule invocation is bounded by its principal's resource limits. PR #669 extended principal scoping into Layer 4, wiring PrincipalProfile constraints into stores, overlay filesystems, and connection handling within the VFS and capsule subsystems. PR #671 capped the sequence at Layer 5 by enforcing capability and group membership checks directly in the kernel and core management API, closing off privileged operations to unauthorized principals. Collectively these changes establish a coherent, defense-in-depth authorization boundary from resource quotas at the invocation level up to administrative API access control.
