Linux FHS directory layout and multi-principal support
PR #537 refactors the directory layout of unicity-astrid to align with the Linux Filesystem Hierarchy Standard (FHS), introducing multi-principal support so that different principals can coexist under a well-structured path hierarchy. PR #540 follows up by updating all crate-level READMEs to document the new FHS-aligned paths, ensuring developers and operators have accurate reference material. Together these changes improve portability, system integration, and long-term maintainability of the Astrid runtime on Linux hosts.
unicity-astrid
OpenClaw Tier 2 plugin install and runtime end-to-end
PR #534 delivers a complete end-to-end implementation of Tier 2 plugin installation and runtime execution within the OpenClaw subsystem. This milestone closes the loop between plugin discovery, installation, and live invocation, enabling Astrid to load and run Tier 2 plugins without manual intervention. The work lays the foundation for a self-contained plugin lifecycle and is a critical step toward a fully operational OpenClaw plugin ecosystem.

Admin UI consistency and independent chain orders
PR #9 refactors the admin UI to enforce visual and behavioural consistency across its components, reducing fragmentation and improving operator experience. PR #8 introduces independent chain orders as a new capability, allowing order entities to exist and be processed autonomously within their own chain context rather than being coupled to a shared sequence. These two PRs together harden the structural integrity of sphere-quest's ordering and administration layers.
sphere-quest
App fetch and cache pipeline in sphere
PR #285 adds logic to fetch and cache application manifests or assets within the sphere service, reducing redundant network calls and improving load times for downstream consumers. Caching app data at this layer means repeated lookups are served locally, which is important for reliability in low-latency or offline-tolerant scenarios. This change establishes a reusable caching pattern that other sphere data types can adopt going forward.
