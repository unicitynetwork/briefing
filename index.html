Astrid2 PRs merged
Security hardening: tighten audit dependency chain
PR #1090 updates the ammonia audit dependency, addressing a security-relevant change in the HTML sanitisation pipeline. Ammonia is a Rust-based HTML sanitiser, and keeping its audit crate current ensures the project stays aligned with vetted, reviewed cryptographic and safety constraints. Stale audit entries can allow unreviewed dependency versions to silently pass cargo-vet or similar supply-chain checks, so this fix closes a potential compliance gap. The change is low-risk but high-importance for maintaining a trustworthy dependency graph.
unicity-astrid
Principal-scoped capsule loading prevents cross-tenant data leakage
PR #1083 introduces isolation of capsule loading by principal, ensuring that one principal cannot inadvertently access or influence capsules belonging to another. Previously, the loading path likely resolved capsules in a shared or insufficiently scoped context, creating a risk of cross-principal data exposure. The fix enforces principal-level scoping at the point of retrieval, which is a critical boundary in a multi-tenant or multi-identity architecture. This change strengthens the core trust model of Astrid and is a meaningful step toward least-privilege data access patterns.

Sphere1 PRs merged
Fix moderation queue pagination response shape mismatch
PR #18 resolves a bug in sphere-backoffice where the moderation queue pagination controls never rendered due to a mismatch between the expected and actual API response shape. The client-side code was likely reading a property such as total or page from a path that did not exist in the server response, causing the pagination component to silently receive undefined values and skip rendering. The fix aligns the response parsing with the real payload structure, restoring full pagination functionality in the moderation interface. This is operationally significant as moderators working through large queues had no way to advance beyond the first page of results.
