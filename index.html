Astrid11 PRs merged
Bootstrap Astrid runtime and capsule-authoring forge skill
PRs #15 and #14 automate what was previously a manual setup step: the Astrid runtime is now bootstrapped automatically as part of the plugin lifecycle rather than requiring operators to pre-install it. Alongside this, PR #14 introduces the /astrid:forge skill, giving Claude a dedicated capsule-authoring capability. Together these changes lower the barrier to entry for Astrid-based workflows significantly.
sage
Fix principal and client ownership bugs in MCP and proxy
PR #951 makes the MCP shim fail loudly when it would silently bind an anonymous principal instead of the explicitly requested one, closing a subtle security gap. PRs #949 and capsule-cli #33 correct ownership of client connect/disconnect events, moving emission to the host rather than the proxy to reflect the correct authority boundary. PR #13 in sage tightens the default for a cleared principal to least-authority claude-code instead of the admin default, and PR #11 heals keyless principals via a bare agent-create in astrid-up.
sage, astrid, capsule-cli
Refactor large files and fix CI OOM under 1000-line cap
PRs #957 and #953 are systematic refactors driven by the project's 1000-line-per-file cap: inline test modules are extracted to sibling files (#957) and host_state.rs is split into #[path] siblings (#953). PR #955 addresses a separate but related build problem — test binaries were retaining debug info and causing linker OOM failures in CI, fixed by stripping debuginfo from test builds.
astrid

Sphere24 PRs merged
Strict token-amount parsing and currency utility cleanup
PR sphere-sdk #612 introduces a breaking change: parseTokenAmount is now strict and a new safeParseTokenAmount variant is added, while the legacy toSmallestUnit helper is removed, eliminating a source of silent currency-handling bugs (tracked as #611). Sphere #383 migrates the wallet layer to consume the new strict API and drops the duplicate currency utility that #612 superseded. The change enforces a single, well-defined code path for all token amount conversions across the stack.
sphere-sdk, sphere
Harden Connect intent handling and approval flow
PRs sphere #381 and #382 address four audit findings (A1–A4, #609): intent params are now validated, the receive intent is implemented, DM auto-approval is properly scoped, and approval permissions are seeded inside an effect rather than during render to avoid React lifecycle hazards. The sphere-connect marketplace skill was updated in unicity-claude-marketplace #9 to reflect corrected intent shapes and the markAsRead scope, bumped to 2.1.0. Connect example #13 fixes protocol version, scopes, error codes, and intent shapes in the reference documentation to match the corrected SDK surface. Sphere-sdk #610 corrects CONNECT.md to match reality and marks the invoicing feature as experimental, closing issue #608.
sphere, sphere-sdk, sphere-sdk-connect-example, unicity-claude-marketplace
Add mint intent with UI, demo, and scope support

Sphere #380 wires up the corresponding wallet UI and bumps the SDK to 0.10.2 as part of the L1-removal wave. Sphere-sdk #606 adds a protocol-surface regression test to guard against future unversioned changes to the Connect API.

Unicity Network9 PRs merged
Testnet2 HAProxy and throughput tuning for 10k req/s
PR bft-terraform #4 upgrades HAProxy hosts from t3a.nano to t3a.small to provide headroom for the load targets defined in bft-ansible #11, which tunes per-shard HAProxy configuration for an aggregate 10,000 requests per second. PR bft-ansible #15 addresses a related problem: the gateway's upstream queue and buffer limits were too small, causing JVM heap exhaustion under backend stalls, fixed by bounding queue depth and raising the JVM heap allocation. Together these changes prepare testnet2 to sustain realistic production-scale traffic without OOM failures at the gateway tier.
bft-ansible, bft-terraform
MongoDB file-descriptor limits, monitoring, and alerting
PR bft-ansible #12 raises mongod's open-file limit from the default 1024 to 64,000, resolving a hard ceiling that was being hit in production. PRs #13 and #14 add the observability layer on top: #13 exposes mongo host file-descriptor counts as a Grafana metric, and #14 introduces a MongoFileDescriptorsHigh alert that fires when fd usage reaches 80% of the nofile limit, providing early warning before the limit is hit again. Bft-terraform #7 migrates all EBS volumes — both data and root — to gp3 across testnet2, improving baseline I/O performance and reducing storage costs.
bft-ansible, bft-terraform
Harden aggregator disk SMT failover and network ID minting
Aggregator-go #161 hardens the disk SMT failover recovery path, improving resilience when the primary disk-backed sparse Merkle tree encounters a fault and the node must recover state. State-transition-sdk-java #72 addresses issue #71 by incorporating the network ID into unicity ID minting, ensuring that IDs generated on different networks are not accidentally treated as equivalent. Both changes improve the correctness and reliability of core node infrastructure on testnet2.
aggregator-go, state-transition-sdk-java
