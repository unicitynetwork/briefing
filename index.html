Unicity Network3 PRs merged
Database Operation Timeouts and Context Propagation
PRs #19 and #20 tackled runaway database operations in the relay service by binding every DB transaction and runtime operation to explicit context timeouts. PR #19 introduced wall-clock time limits on DB transactions alongside a pprof endpoint to diagnose a goroutine leak that was allowing blocked DB calls to accumulate indefinitely. PR #20 extended this work by ensuring every individual runtime DB operation is also wrapped with a context timeout, closing the remaining exposure surface identified in issue #18. Together these changes prevent the relay from exhausting goroutine or connection-pool resources under slow or stalled database conditions, making the service significantly more resilient in production.
unicity-relay
Extended OR-Disjunction Lookup Statistics (m007)
PR #18 added m007 extended statistics instrumentation to the tokens-relay, specifically targeting OR-disjunction event lookup paths as tracked in issue #17. The change surfaces per-query metrics for disjunctive predicate resolution, giving operators visibility into a query pattern that was previously opaque in monitoring dashboards. This matters because OR-disjunction lookups can exhibit non-linear performance characteristics, and without dedicated metrics they were difficult to profile or alert on. The fix establishes a baseline for future performance tuning and anomaly detection on this code path.
