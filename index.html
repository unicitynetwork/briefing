Astrid54 PRs merged
SDK 0.5.x release train: macros and tool describe fixes
Three SDK releases landed today: 0.5.0, 0.5.1, and 0.5.2 (PRs #18, #20, #22). PR #19 fixed the tool_describe macro to return JSON bytes directly for hooks::trigger compatibility, and PR #21 followed up to wrap the return value in Ok() for FnResult compatibility. These macro correctness fixes were critical enough to warrant two rapid patch releases, ensuring capsule authors can reliably expose tool metadata through the hook system.
sdk-rust
IPC tool dispatch and hook-based tool collection across capsules
A coordinated set of PRs added IPC interceptor entries for tool dispatch to capsule-system (#1), capsule-skills (#8), capsule-http (#6), capsule-fs (#7), and capsule-shell (#5), establishing a uniform interception layer for routing tool calls. capsule-prompt-builder (#6) was updated to collect tools via trigger_hook and fetch session messages, while capsule-react (#9) was refactored to receive both tools and messages from the prompt builder. Together these changes wire up the full tool-dispatch pipeline from capsule registration through prompt assembly. capsule-system (#2) also gained automatic installation of the capsule-development skill on capsule install, streamlining the developer onboarding experience.
capsule-system, capsule-skills, capsule-http, capsule-fs, capsule-shell, capsule-prompt-builder, capsule-react
VFS scheme rename workspace:// to cwd:// and memory path fixes
The virtual filesystem scheme workspace:// was renamed to cwd:// across the core runtime and affected capsules via astrid #599, capsule-fs #8, capsule-identity #10, and capsule-skills #9, clarifying that the scheme represents the current working directory context. capsule-memory #6 fixed an earlier bug where memory.md was not using the VFS scheme at all, and #7 extended it to read the project folder name from the cwd_dir env config. capsule-identity #11 corrected spark.toml to be stored under home:// rather than cwd://, preventing identity data from leaking into project directories. These changes establish clearer separation between user-home and project-scoped virtual paths.
astrid, capsule-fs, capsule-identity, capsule-skills, capsule-memory
Astralis composition, agents capsule bootstrap, and sdk 0.5.2 rollout
astralis received two composition PRs (#3 and #4) iterating on the addition of the agents capsule and adjusting cwd_dir handling for memory and identity, with #4 finalising the removal of cwd_dir from identity. A new capsule-agents repository was bootstrapped with an env field name fix (PR #2) alongside capsule-memory #10 receiving the same Capsule.toml correction. capsule-memory #8 added injection of both personal and project memory, enabling context-aware memory scoping per session. Following the sdk-rust 0.5.2 release, 17 capsule repositories received coordinated dependency bump PRs, and astrid #597 fixed WIT installation to target the principal home directory rather than a root wit/ folder. The awesome-astrid community highlights list was also bootstrapped via PR #1 by @Copilot, establishing a community curation resource for the ecosystem.
