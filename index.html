Astrid20 PRs merged
Rust SDK 0.4.0: typed APIs, std alignment, Error impls
PRs #10, #12, #14, and #15 collectively deliver the sdk-rust 0.4.0 release. PR #14 is the centrepiece, aligning the SDK with Rust std conventions: typed fs::read_dir and fs::metadata returns, SystemTime from time::now(), Display-based logging, and proper Error trait implementations. PR #12 introduces an mpsc-style channel API with recv, send, try_recv, and try_accept, replacing the previous streaming subscription model. PR #10 adds #[capsule(state)] macro support, SysError dispatch, and a stream-closed sentinel. The 0.4.0 tag in #15 cuts the release, making these capabilities available to all downstream capsule authors.
sdk-rust
Capsule ecosystem migrated to new SDK typed APIs
Eight capsule repositories were updated to consume the new SDK 0.4.0 surface area. PRs #4 in capsule-http, #5 in capsule-openai-compat, and #3 in capsule-anthropic migrate network code to the typed http::Request/Response API. PRs #5 in capsule-skills and #4 in capsule-fs adopt the typed fs::read_dir and fs::metadata APIs, while capsule-react #5 switches to the SystemTime-returning time::now(). capsule-cli #4 moves to the mpsc-style net API and removes the now-obsolete stream.delta subscription. capsule-identity #4 becomes fully self-contained with KV-backed state, an onboarding flow, and import/export of identity credentials.
capsule-http, capsule-openai-compat, capsule-anthropic, capsule-react, capsule-skills, capsule-fs, capsule-cli, capsule-identity
Astrid host: net wire format, IPC ordering, ephemeral shutdown
PRs #521 through #532 address stability, correctness, and lifecycle management in the Astrid host layer. PR #532 replaces the raw sentinel value in the network wire format with a structured NetReadStatus enum, making stream-end signalling explicit and type-safe. PR #529 fixes a WASM trap triggered when a headless client disconnects mid-stream due to a broken pipe. PR #527 introduces ephemeral shutdown when the last client disconnects and renames the internal plugin abstraction to capsule throughout the host layer. PR #525 corrects per-capsule IPC dispatch ordering and attaches a monotonic sequence number to every IpcMessage, preventing out-of-order delivery bugs.
astrid
Astrid host: boot-time setup and environment stability fixes
PRs #521 and #523 address two operational reliability gaps in the Astrid host. PR #523 ensures the ~/.astrid/shared/ directory is created on boot, preventing runtime failures in capsules that rely on shared storage being present. PR #521 preserves an existing .env.json file across capsule reinstalls, protecting user-configured environment variables from being overwritten, and raises the WASM execution timeout to five minutes to accommodate longer-running capsule initialisation. Together these changes reduce friction during development and deployment cycles where capsules are frequently reinstalled or updated. Both fixes are low-risk and targeted with no API surface changes.

Sphere2 PRs merged
Direct message filtering and memo achievements
Two PRs extend Sphere's messaging and engagement capabilities. sphere-sdk #93 adds a dmSince option, enabling clients to fetch direct messages from a specific point in time rather than retrieving the full history, which reduces payload size for incremental sync scenarios. sphere-quest #7 introduces memo achievements, tying quest reward logic to memo-based actions within the platform. Together these changes broaden the event surface that the quest and reward system can respond to, encouraging messaging engagement as a first-class achievement trigger.
sphere-sdk, sphere-quest

Relay retention policy, openclaw init, and split logic refactor
Three PRs advance infrastructure and SDK correctness across the unicitynetwork org. unicity-relay #6 implements a Message Retention Policy, defining how long messages are stored in the relay before expiry, which is critical for managing storage growth and ensuring predictable delivery semantics. openclaw-unicity #7 delivers init improvements, hardening the bootstrap sequence for the openclaw integration layer. state-transition-sdk #100 resolves issue #95 by changing the split logic to return only split metadata rather than full split content, reducing unnecessary data transfer and clarifying the return contract for downstream consumers.
