Sphere20 PRs merged
Fail-loud network config: no silent defaults
PRs #467–#471 (sphere-sdk) and #341–#342 (sphere) implement a three-phase hardening of network configuration. Phase 1 (#467) added regression guards for the network-drop bug class. Phase 2 (#469) removed silent testnet/mainnet fallbacks so missing network config now throws immediately. Phase 3 (#471) made NetworkConfig a required parameter and added a startup consistency assertion. Sphere itself was updated in #341 to pass network through Sphere.import and importFromLegacyFile, and bumped to SDK dev.4/dev.5 accordingly. This eliminates an entire class of silent misconfiguration bugs where the wrong network was used without any error.
sphere-sdk, sphere
Registry and token symbol fixes across networks
PR #466 (sphere-sdk) forwards the network parameter through the init→create/load chain and introduces a per-network cache, fixing testnet2 icon and mint resolution. PR #340 (sphere) passes network into Sphere.init so the registry correctly loads testnet2 assets, resolving an icons regression. PR #338 (sphere) fixes token symbol resolution for fresh wallets without requiring a re-login. Together these changes ensure multi-network deployments see correct registry data from first launch.
sphere-sdk, sphere
Payments, swap, and profile correctness fixes
PR #453 splits the payments flow into separate local-commit and remote-publish phases, addressing issue #444 and reducing the risk of partial state on failure. PR #451 corrects v2 send gating to check the recipient's chainPubkey rather than DirectAddress. PR #461 ensures terminal swaps are included in resolveSwapId/getSwaps and adds write-side error codes (#447). PR #459 adds fail-fast behaviour on a missing transportPubkey (#457), and PR #452 bounds the pre-shutdown publish-retry spin to prevent indefinite loops on profile shutdown (#450).
sphere-sdk
Transport, mux, profile and escrow reliability
PR #465 fixes an await gap in the mux dispatch path that could cause dropped messages under concurrent transport use (#464). PR #460 asserts that the mux subscription gate remains closed during module load, preventing premature message delivery (#448). PR #463 addresses HIGH-severity findings from a post-merge review of the profile PR #453. PR #458 documents the escrow DIRECT address fallback for nametag resolution (#456), and PR #462 investigates single-coin faucet flakiness in the UXF suite (#455). PR #339 disables Group Chat by hiding the desktop icon and blocking access routes.

H2C (HTTP/2 cleartext) end-to-end for testnet2

Together these PRs complete the h2c upgrade path for the testnet2 aggregator tier.
aggregator-subscription, bft-ansible
Java SDK: networkId verification and mint param reorder
PR #69 resolves issue #68 by adding a networkId field to the Java state-transition SDK and introducing verification logic to ensure transactions are bound to the correct network. Mint transaction parameters are also reordered for consistency with the wider protocol specification. This brings the Java SDK into alignment with network-awareness requirements seen across other Unicity components and prevents cross-network transaction replay.
