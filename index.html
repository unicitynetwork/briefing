Admin UI Migrated to Sphere-Quest
PR #26 refactored the admin interface as part of a broader Sphere UI migration effort, moving admin functionality into the sphere-quest codebase. PR #25 brought the branch up to date with main to prepare for clean integration. Together these changes consolidate the admin surface under a unified UI layer, reducing fragmentation and simplifying future maintenance.
sphere-quest
Credential and Dialog Bug Fixes in Sphere-Quest
PR #27 dropped a legacy platform_1 unique index that was blocking per-project credential management, resolving a database-level constraint issue left over from an earlier schema. PR #28 addressed edge cases in confirm dialog handling, specifically improving behaviour around pending states and 204 no-content responses. These fixes harden the reliability of the credentialing flow and prevent silent failures when the server returns no body.
sphere-quest
dApp Logout Notification Fix in Sphere Wallet
PR #290 fixed a bug where connected dApps were not being notified when the wallet session was logged out inside the Connection Manager. Without this fix, dApps could remain in a stale authenticated state after a user signs out, creating both UX and security inconsistencies. The change ensures the logout event is properly broadcast to all connected dApp listeners.
sphere
Apps Registry Updated
PR #1 updated apps.json, the central registry that defines which applications are surfaced within the Sphere ecosystem. This foundational change ensures the app discovery layer reflects the current set of supported or promoted dApps. Keeping this manifest accurate is critical for users and integrators relying on the Sphere app catalogue.
