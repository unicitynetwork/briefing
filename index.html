Astrid1 PRs merged
Hot-load capsules into live daemon without restart
PR #959 adds CLI support for hot-loading a freshly-installed capsule directly into a running daemon process. Previously, installing a new capsule would require a daemon restart to pick up the new module. This change enables zero-downtime capsule deployment, which is critical for production environments where daemon continuity matters.
unicity-astrid
Sphere36 PRs merged
Media uploader: upload/URL toggle, validation, auto-fit
A multi-PR effort by @KruGoL (#11–#16 across sphere-ui, sphere-backoffice, sphere-dev-portal) builds out a fully-featured MediaUploader component. Key additions include pixel-dimension validation on the client (#11), auto-fit center-crop and downscale instead of rejecting oversized images (#14), a deferred-upload mode for MediaGallery (#12), and a source toggle allowing either file upload or URL input (#15). Screenshots and videos are now persisted on project settings save in the dev portal (#13). The backoffice and dev portal were bumped to sphere-ui 0.1.24 and 0.1.27 to consume each increment, and the admin create form now exposes logo, banner, screenshot, and video uploaders (#13 backoffice, #15 dev-portal).
sphere-ui, sphere-backoffice, sphere-dev-portal
HTTPS enforcement and deep-link URL normalization
PR #35 (sphere-api) adds server-side validation that enforces https:// on all project website and app URLs, with matching client-side enforcement added in sphere-backoffice #10 and sphere-dev-portal #11. PR #385 in the sphere monorepo fixes the custom-agent deep link handler to normalize the scheme and adds a dev SPA fallback, preventing malformed ?url= parameters from reaching the runtime. A .gitattributes fix (#36, sphere-api) forces LF line endings on shell scripts, resolving a Windows autocrlf issue that was breaking the localstack bootstrap. Together these close a surface of subtle environment and input-validation bugs.
sphere-api, sphere-backoffice, sphere-dev-portal, sphere
Connect network: base-unit amounts, error handling, SDK 0.10.3 rollout
PR #613 (sphere-sdk) clarifies that send and payment_request amounts in the Connect protocol are base units (U1, tracked in #609), with the connect example updated in #14 to use base units in practice. PR #384 (sphere) improves the Connect intent lifecycle by rejecting the intent on send failure, surfacing human-readable errors, and removing dead prefill code. sphere-sdk 0.10.3, which drops the removed L1 surface and adds Connect network support, was rolled out across sphere-api (#34), sphere-backoffice (#9), sphere-dev-portal (#10), sphere-quest-frontend (#11), sphere-sdk-connect-example (#24), and agentic-chatbot (#12). The agentic-chatbot migration (#12) also moves bots to wallet-api v2 on testnet2.
sphere-sdk, sphere, sphere-sdk-connect-example, sphere-dev-portal, sphere-backoffice, sphere-api, sphere-quest-frontend, unicity-claude-marketplace, agentic-chatbot
Soak test reliability: SIGPIPE race and JSON decode fixes
Two soak-test failures in the trader-roundtrip suite were diagnosed and fixed by @vrogojin. PR #615 resolves a pipefail+SIGPIPE race in §4 (ACP probe) that was masking every success as a timeout, while PR #616 fixes §10 (portfolio_balance) to handle trailing log lines using JSONDecoder.raw_decode instead of a strict json.loads call. PR #51 (sphere-cli) opts out HMCP and ACP sendDM from self-wrapping, fixing transport issue sphere-sdk#559, with PR #614 re-exporting SendMessageOptions from the top-level index to restore the public API surface. PR #582 adds diagnostic instrumentation to the Mux dedup hydration and hit path to aid future transport debugging.
sphere-sdk, sphere-cli
Wallet inventory: fix self-send token loss on claim race
PR #71 (wallet-api) fixes a race condition in the inventory self-send path where a token could be lost when the self-claim won the race against the incoming transfer, tracked as issue #70. The fix ensures the token is correctly retained in the sender's inventory regardless of claim ordering. This closes a correctness bug that could silently drop tokens in loopback transfer scenarios.
wallet-api

Testnet2 RocksDB SMT: persistent storage and EBS backups
PR #17 (bft-ansible) deploys the on-disk RocksDB SMT build (sha-fa709d2) to testnet2 aggregators, replacing the in-memory SMT, and adds alerting for disk-SMT health and disk-fill conditions. PR #8 (bft-terraform) provisions per-node EBS data volumes for both the aggregator RocksDB store and the BFT root directory, with daily EBS snapshot backups configured. Together these changes move testnet2 to a durable, production-grade storage topology and establish the backup baseline required before mainnet-scale load.
bft-ansible, bft-terraform
Aggregator metrics and staging deploy fixes
PR #163 (aggregator-go) corrects the persisted-nodes Prometheus gauge, which was reporting inaccurate values and undermining observability of node persistence state. PR #71 (semanticd) fixes the e2e-smoke test invocation to run from the compose-dir root, unblocking the staging deploy pipeline that was broken by an incorrect working-directory assumption. These are targeted reliability fixes that restore accurate monitoring and continuous deployment to the testnet2 environment.
aggregator-go, semanticd
Dashboard policy edit support
PR #70 (semanticd) implements policy edit support for the dashboard, tracked under TK-260616. This extends the dashboard's capability from read-only policy inspection to active policy management. The change enables operators to modify policies directly through the dashboard interface without requiring manual backend intervention.
