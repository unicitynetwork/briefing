Nostr SDK Reliability: Ping, CLOSED, and Query Timeouts
PRs #5 and #8 jointly address three related reliability gaps in the Nostr SDK layer across the Go and JavaScript implementations. PR #5 fixes the ping firehose — preventing runaway keepalive traffic — and adds proper handling of CLOSED frames alongside silent query timeout surfacing in the Go SDK. PR #8 mirrors these fixes in the JS client, scoping keepalive pings to avoid broadcast storms and ensuring CLOSED frames are correctly recognised and propagated, while also making query timeouts visible to callers rather than failing silently. Together these changes significantly improve connection stability and debuggability for any service consuming Nostr relays through the Unicity SDK stack.
nostr-sdk, nostr-js-sdk
CBOR Encoding Fix in State Transition SDK
PR #114 resolves issue #113, correcting CBOR serialisation bugs in the JavaScript State Transition SDK. The key change replaces the use of the raw user predicate with the encoded predicate form, ensuring that CBOR round-trips produce well-formed, standards-compliant byte representations. Malformed predicates in prior serialisation could cause silent data corruption or validation failures downstream in the state transition pipeline. This fix is important for correctness of any state proof or transition that carries predicate data across network or storage boundaries.
state-transition-sdk-js
SIF Dashboard: Aspirational UI Scaffolding
PR #32 introduces an aspirational dashboard for the SIF (sif-dash) component, contributed by @jvsteiner. This PR lays out the intended end-state UI for SIF monitoring and interaction within the Unicity network, serving as both a design reference and a functional scaffold for future feature work. Aspirational dashboards of this kind help align engineering and product on target capabilities before full backend integration is complete.
