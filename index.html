Unicity Network8 PRs merged · 7 new branches
SSL Manager Alias Proxy Stability and Crash Recovery
A focused burst of four fixes hardened the alias proxy component of ssl-manager: #6 added auto-restart on crash, #7 wrapped the proxy in a bash supervisor to handle SIGSEGV crashes specifically, #8 introduced a 10-second TLS handshake timeout to prevent thread exhaustion caused by health check traffic, and #11 ensured the alias proxy correctly forwards to the primary HTTPS port for full API access. Each fix addressed a distinct failure mode observed in production-like conditions, collectively making the alias proxy significantly more resilient. The corresponding branches (fix/alias-proxy-auto-restart, fix/alias-proxy-shell-supervisor, fix/alias-proxy-handshake-timeout) were all created and merged same-day, reflecting rapid iterative hardening.
ssl-manager
Generic Nginx Alias Block Generation in SSL Manager
SSL-manager #9 refactored alias registration to use the app HTTPS port and removed the legacy TLS proxy startup path, simplifying the service lifecycle. Building on that, #10 introduced a feature to automatically generate generic nginx alias blocks via ssl-setup, driven by the new branch 'feat/generic-nginx-alias-blocks'. This means alias configuration is now templated and emitted programmatically rather than maintained by hand, reducing toil and configuration drift for any service that registers an alias.
ssl-manager
IPFS Storage Migrated to SSL Manager Nginx Aliases
Two refactor PRs updated ipfs-storage to consume the new ssl-manager nginx alias infrastructure: #4 ('refactor/nginx-alias-blocks') replaced the previous TLS proxy approach with nginx alias server blocks, and #5 ('refactor/use-ssl-manager-alias-nginx') went further to adopt the generic alias nginx config now produced by ssl-manager. This removes a bespoke TLS proxy layer from ipfs-storage, aligning it with the centralised alias pattern and reducing duplicated nginx configuration. The change ensures ipfs-storage benefits from the same crash recovery and timeout protections hardened in ssl-manager on the same day.
ipfs-storage
