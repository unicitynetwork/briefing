WIT Contract Alignment Across Kernel and Capsules
PRs #794, #789, and #21 correct the kernel and capsule-cli to recognise and emit the canonical client.v1.connect / client.v1.connected / client.v1.disconnect topics as defined in the WIT contract. PR #51 in sdk-rust fixes the accompanying doc comment on the random-bytes binding to match the WIT specification. PR #801 closes a related correctness gap by sourcing capsule random-bytes from the OS CSPRNG rather than a weaker fallback. Together these changes ensure the full connection-lifecycle signalling path is contract-compliant end-to-end.
astrid, capsule-cli, sdk-rust
Gateway Observability: Metrics, Histograms, Logs, and Audit API
PRs #798, #797, #787, #780, and #781 significantly expand runtime visibility into Astrid. The /metrics endpoint now exposes process-collector data, astrid_build_info, event-bus saturation gauges, and daemon saturation metrics (#798, #797). Per-request structured logging and latency histograms were added (#780), and a new GET /api/sys/audit endpoint enables historical audit queries (#781). Passive event-bus storm diagnostics (#787) provide early-warning detection of message floods without active intervention, rounding out a comprehensive observability layer.

Gateway TLS, OpenAPI Typing, and Operator Runbook

PR #782 delivers a deployment runbook and operator documentation to support production rollouts. These changes collectively bring the gateway to a production-ready baseline.

Capsule Ecosystem: Locked Builds, WIT Migration, and Singleton Guard
Fourteen capsule repositories received Cargo.lock commits and --locked build enforcement (chore PRs across capsule-openai through capsule-prompt-builder), ensuring fully reproducible, hermetically sealed CI builds across the ecosystem. capsule-users (#3) and capsule-system (#8) were migrated to per-domain WIT interfaces and the wasm32-unknown-unknown target, decoupling them from the legacy monolithic WIT bundle. PR #792 in astrid replaces the previous singleton guard with a robust flock-based lock combined with a fail-closed socket path, preventing duplicate daemon instances under adverse conditions.

Security Audit #333: High and Critical Vulnerability Fixes
Seven PRs directly address findings from Security Audit #333: H1 through H5 and C1 through C3. Critical fixes include plugging a plaintext-seed exposure window in encrypt() (#346, C1), forcing a durable flush before profile migration cleanup (#347, C2), and adding a probe before destructive oplog resets (#361, C3). High-severity fixes cover the conservative-sender source lock (#349, H1), manifest tokenId binding to root content (#351, H2), mergePkg skipped-token surfacing with strict mode (#352, H3), requestId binding-gate re-derivation (#353, H4), and source-token unlock on failed-permanent transfer state (#354, H5). A recompute-content verifier was added to ManifestCas (#355, H7), and bundleCid envelope timestamp is now locked across retry attempts to ensure determinism (#362).
sphere-sdk
Profile Durability, Sync Correctness, and Page-Freeze Stability
PRs #340 and #334 address profile snapshot reliability by introducing SingleBlobSyncWriter for tombstones and invalidatedNametags (#340) and hardening GroupChat fan-out, CBOR handling, and snapshot-epoch logic under soak conditions (#334). PR #165 closes at-least-once invariant gaps and resolves cross-device sync edge cases. On the browser side, PRs #326 and #334 fix page-freeze teardown by disabling the deprecated IPFS sync default and properly handling pagehide events, preventing data loss on tab suspension. A false-negative in the aggregator pinger when round equals zero was also corrected (#342).
