Inclusion Proof Format Upgraded Across Stack
PRs #108 (state-transition-sdk-js) and #141 (aggregator-go) jointly deliver the new v2 inclusion proof format across both the JavaScript SDK and the Go aggregator. PR #141 implements the v2 tree structure and corresponding inclusion proof serialization on the aggregator side, while PR #108 updates the client-facing SDK to match the revised format as tracked in issue #105. Together these changes ensure that proof generation and verification remain in sync across language boundaries. This is a breaking-protocol upgrade that improves the structural integrity and verifiability of state transition proofs across the Unicity network.
state-transition-sdk-js, aggregator-go
Finality Gadget CI and Code Quality
PRs #15 and #13 bring foundational engineering hygiene to the finality-gadget repository. PR #15 introduces a GitHub Actions CI pipeline, enabling automated build and test validation on every push and pull request going forward. PR #13 addresses review feedback with targeted fixes, tightening code quality ahead of broader integration. Having CI in place now means the finality gadget can scale contribution velocity without sacrificing correctness. These two PRs together mark the component as ready for more active collaborative development.
finality-gadget
Prometheus Metrics and Grafana Cloud Observability
PR #2 in the unicity-tokens-relay repository adds Prometheus metrics instrumentation alongside Grafana Cloud integration. This gives operators real-time visibility into relay behaviour through standardised metric endpoints that Grafana Cloud can scrape and visualise. Being only the second merged PR in this repository, the change establishes observability as a first-class concern from the very start of the relay's lifecycle. Monitoring infrastructure at this early stage will accelerate debugging and capacity planning as the relay moves toward production traffic.
unicity-tokens-relay
