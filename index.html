Astrid34 PRs merged
Streaming HTTP airlock added to capsule runtime
astrid #509 adds a handle-based streaming HTTP airlock with three new host functions (astrid_http_stream_start, astrid_http_stream_read, astrid_http_stream_close), letting capsules consume HTTP responses chunk-by-chunk instead of buffering entire bodies. sdk-rust #4 adds the companion SDK surface — HttpStreamHandle plus stream_start/stream_read — so capsule authors can use the feature from Rust. sdk-rust #5 ships this as the 0.2.2 release. This unblocks real-time LLM streaming responses and large-payload handling inside capsules.
astrid, sdk-rust
Daemon/CLI split, boot pipeline fixes, headless mode
astrid #507 extracts the kernel daemon and capsule build tooling into standalone binaries (astrid-daemon, astrid-build), making the CLI a pure socket client with new lifecycle commands: start, status, stop. astrid #513 follows up with end-to-end boot pipeline bug fixes including a KV double-encoding fix in kv_get_impl, plus headless mode and post-install onboarding flow. astrid #512 bumps the full workspace from 0.3.0 to 0.4.0 to ship these changes. Together these make the daemon independently deployable and the CLI fully decoupled.
astrid
SDK 0.3.0: doc-comment tool descriptions and mutable flag
sdk-rust #7 extends the #[capsule] proc macro so that /// doc comments on #[astrid::tool] methods become tool descriptions in the exported JSON schema, and adds an inline mutable flag. sdk-rust #8 releases this as 0.3.0, noting a breaking schema change: the export format moves from a plain tools map to { tools: {...}, description: '...' }. Six capsules (openai-compat, identity, skills, shell, http, fs) each land companion PRs adding doc comments so LLMs receive meaningful descriptions when selecting tools. All remaining capsules are simultaneously bumped to sdk 0.3.0, with uuid rng-getrandom and wasm32-wasip1 CI fixes applied where needed.
sdk-rust, capsule-openai-compat, capsule-identity, capsule-skills, capsule-shell, capsule-http, capsule-fs
Capsule housekeeping: duplicate tools and dead IPC subscription removed
capsule-skills #4 removes duplicate [[tool]] entries from Capsule.toml that were causing each skill tool to register twice in the kernel's tool registry, a silent correctness bug visible via astrid capsule list-tools. capsule-cli #3 drops the agent.v1.stream.delta IPC subscription, a dead topic that no longer exists on the event bus and was adding unnecessary overhead to every CLI proxy session. Both are small targeted fixes that clean up observable misbehaviour introduced during earlier development.
capsule-skills, capsule-cli
