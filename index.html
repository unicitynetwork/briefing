Automated AOS Release Polling via Homebrew CI
PR #6 introduces a CI pipeline that polls for newly published stable AOS releases, enabling the homebrew-tap to track upstream releases automatically rather than requiring manual intervention. This reduces the operational overhead of keeping the homebrew formula current and ensures end-users on macOS receive timely updates through the standard brew upgrade workflow. The automation closes a gap in the release distribution chain for the AOS binary.
unicity-aos / homebrew-tap
Sphere5 PRs merged
Eliminate Double-Pay Bugs in Payment Flows
PRs #681 and #682 together address three confirmed double-payment incidents (issues #676 and #677). PR #681 prevents duplicate charges arising from concurrent send and resume operations, while PR #682 defers the emission of the payment-request 'paid' event until the underlying transfer has fully completed, eliminating a race condition that could trigger a reload-induced second charge. These fixes harden the payment state machine against re-entrancy and asynchronous timing hazards that were reachable in normal user flows.
unicity-sphere / sphere-sdk
Durable suspectedSpent Demotion in Wallet Custody
PR #680 (SPHERE-4) makes the suspectedSpent demotion operation durable within the wallet-api custody layer. Previously, a crash or restart between demotion and confirmation could leave a coin in an inconsistent suspected-spent state, risking funds being treated as unavailable or triggering erroneous double-spend logic. The fix persists the demotion atomically so that recovery paths in the custody layer always reflect the correct coin status.
unicity-sphere / sphere-sdk
Migrate Crypto Layer from elliptic to @noble/curves
PR #675 replaces the legacy elliptic library with the modern @noble/curves package across the sphere-sdk, addressing issue #674. The noble suite is actively maintained, has undergone independent audits, and provides a smaller, tree-shakeable API surface with no transitive native dependencies. PR #437 in the sphere frontend then removes the now-dead elliptic and crypto-js dependency declarations, completing a clean sweep of the deprecated crypto stack and reducing bundle size.
unicity-sphere / sphere-sdk, sphere
