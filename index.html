Sphere5 PRs merged
Subscription quota gating with 429/401 upgrade routing
PR #404 completes Phase 4 of the subscription enforcement rollout by introducing a dual-mode quota gate: a proactive pre-send check that blocks outbound messages before they hit the network, and a reactive path that intercepts 429 and 401 responses and routes them through the upgrade flow. This prevents wasted sends against exhausted quotas and gives users a direct upgrade prompt rather than a raw error. PR #402 accompanies this by aligning the wallet-side logic with the live SGW gateway contract, ensuring the client speaks the correct protocol version against the production gateway. Together these two PRs close the gap between sandbox behaviour and the live subscription enforcement surface.
unicity-sphere
Single-source Grafana dashboards and alert rules
PR #28 restructures the wallet-api Grafana dashboard so that the dashboard JSON and alert definitions are authored once inside the application repository and pushed out to infrastructure, eliminating drift between the two. PR #27 adds a developer guide documenting the end-to-end workflow for updating that dashboard and its alert rules so the process is repeatable without tribal knowledge. PR #93 wires two newly introduced intent metrics into the live dashboard and attaches alert rules so operators gain immediate visibility into intent processing behaviour. Collectively these three PRs establish a single source of truth for observability configuration and reduce the operational burden of keeping dashboards in sync with application changes.
