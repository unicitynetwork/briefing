Subscription key architecture: per-address and wallet-level
PR #406 changes the default subscription key model so that each address receives its own independent key, with an opt-in inheritance path for wallets that prefer a shared key, and ships accompanying UI polish. PR #405 establishes the complementary wallet-level key strategy using the index-0 identity, allowing individual addresses to opt out of the shared key. The two PRs together implement a complete, layered key hierarchy for subscriptions, enabling fine-grained privacy control without breaking existing wallet behaviour.
sphere
Sentry error reporting with wallet-grade scrubbing
PR #407 integrates Sentry into the sphere application in errors-only mode, applying wallet-grade data scrubbing before any event is transmitted to ensure private key material and address data never leave the client. PR #15 bumps the agentic-chatbot bots dependency to sphere-sdk 0.11.4, keeping the chatbot runtime aligned with the latest SDK release. The Sentry integration gives the engineering team production observability without compromising user privacy.
