Astrid18 PRs merged
Multi-tenant agent lifecycle with secrets and config inheritance
PRs #733 and #731 establish a first-class secret store, introducing a file-per-secret layout for env_type=secret environment variables alongside a new EnvScope {Agent, Shared} operator-only sharing model from #715. Agent creation now inherits default config, KV, and secret files automatically (#733), and the CLI chains quota, capability, and group provisioning in a single agent create flow (#729 from F-C). Together these changes mean a newly created agent arrives fully configured without manual follow-up steps, which is critical for reproducible multi-tenant deployments.
unicity-astrid
Capability and group access control hardened end-to-end
PR #725 (F-B) adds admin.agent.modify for group membership, while #727 ensures caps check correctly resolves group-inherited capabilities rather than only direct grants. The breaking change in #721 (F-E) enforces an --unsafe-admin rail before any wildcard '*' capability grant, preventing accidental privilege escalation. PR #737 self-scopes GroupList so the capability check functions correctly for non-admin callers, closing an F-D regression. Collectively these PRs close several privilege-boundary gaps that would have been reachable by unprivileged agents in a shared kernel.
unicity-astrid
Principal propagation through IPC, capsules, and uplinks
PR #719 threads the active principal through uplinks at the kernel level, and #735 propagates the publisher principal through ipc::recv so non-default chat channels no longer hang waiting for attribution. The WIT repository received eleven interface additions and extensions in #2, providing the typed surface area that makes principal-aware IPC possible across language boundaries. PR #709 hardens deserialization by defaulting IpcMessage timestamp and signature fields, preventing panics on legacy messages. These changes establish a coherent identity chain from capsule publish to kernel routing to subscriber receipt.
unicity-astrid
CLI usability: session UUIDs, who attribution, and capsule ergonomics
PR #741 lets --session accept either a UUID or a human-readable name with auto-detection, removing a common source of operator error. Per-agent connection attribution in astrid who is fixed in #739, so the command now correctly maps connections to the agent that owns them rather than collapsing them. PR #717 provisions agent home directories automatically and drops the now-redundant --agent flag from capsule subcommands. PR #723 fixes capability display formatting, adds cap-grammar help text, and corrects persistent-mode idle defaults, rounding out the day-to-day CLI experience.
