Ledger & Payout Correctness Fixes
PRs #121 and #119 address two critical correctness gaps in the accounting and swap layers. #121 ensures the provisional ledger is fully flushed before payInvoice and returnInvoicePayment return, eliminating the risk of stale state being observed by callers. #119 introduces an explicit OVER_COVERAGE rejection path in verifyPayout, replacing an implicit fallthrough that could allow over-covered swap payouts to pass validation. Together these changes harden the financial logic against subtle ordering and boundary-condition bugs surfaced during live end-to-end testing.
sphere-sdk
Race Condition Elimination from Live E2E Sessions
PR #115 bundles a set of swap and accounting race-condition fixes identified directly from live end-to-end runs. The changes address concurrency hazards where interleaved async operations could leave swap state or accounting records inconsistent. Catching these issues in a live environment rather than unit tests reflects the complexity of timing-dependent ledger interactions. The fix set is grouped as a single PR to ensure the corrections are applied atomically and remain bisectable as a unit.
sphere-sdk
Nostr Transport Timeout Tuned for Real Networks
PR #118 bumps the NostrTransportProvider query timeout from 5 seconds to 20 seconds. The original 5s threshold was too aggressive for real-world Nostr relay latency, causing premature timeouts during live e2e sessions. Extending to 20s aligns the transport layer with observed network conditions without sacrificing failure detection. This is a targeted, low-risk change that improves reliability of the SDK's default transport configuration.
sphere-sdk
CLI & SDK Interface Alignment with Trader Service
PRs #8 and #6 correct interface mismatches between the CLI layer and the underlying SDK and trader-service. #8 aligns the ACP wire shape in the trader CLI with the trader-service contract, fixing field discrepancies in expiry, volume, and drop status that would cause silent data mismatches at runtime. #6 updates the legacy CLI to import from the correct sphere-sdk L1 namespace and adds previously missing exports, restoring compilation and runtime correctness for legacy entry points. These fixes are essential for consistent behaviour across all CLI surfaces as the SDK namespace has evolved.

Redis Sentinel HA Support for Aggregator
PR #149 adds Redis Sentinel support to the aggregator-go service, enabling high-availability Redis deployments as a drop-in alternative to single-node Redis. Sentinel mode provides automatic primary election and failover, removing the aggregator's Redis connection as a single point of failure. The change is significant for production readiness, allowing operators to run georedundant Redis topologies without modifying aggregator business logic. This brings the aggregator's infrastructure dependencies in line with HA requirements expected for a live network deployment.
