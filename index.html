Concurrency money-safety: forward-complete legs and remainder surfacing
PR #690 is the fourth instalment of the ongoing concurrency audit, targeting financial safety around multi-leg payment flows. It ensures that forward-completed legs are correctly resumed and that any remaining balance is surfaced to the caller rather than silently dropped. This closes a class of race-condition bugs where partial completions could leave funds in an indeterminate state. PR #691 complements this by fixing construction rot in three end-to-end test files, forwarding options.network properly into Sphere.init so the e2e suite accurately reflects real network configurations. PR #445 then bumps sphere-sdk to 0.11.15 in the sphere consumer repo, propagating both fixes downstream.

PII/NER detection pipeline and enterprise RBAC/SSO
PR #78 delivers an end-to-end dlp_ner model integration into semanticd, wiring a PII and named-entity recognition model into the detection pipeline and hardening the pipeline itself against edge-case failures. This gives the platform native data-loss-prevention capabilities without requiring an external scanning service. PR #77 introduces enterprise-grade access control by adding RBAC and SSO support via OIDC and SCIM protocols to both the Management API and the dashboard. Together these two features position semanticd for enterprise deployment scenarios where compliance, identity federation, and fine-grained role management are mandatory requirements.
