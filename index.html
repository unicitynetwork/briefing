Unicity Network8 PRs merged
Generic Nginx Alias Block Generation via ssl-manager
PRs #10 and #9 in ssl-manager introduce a generic nginx alias block generation system baked into the ssl-setup process, replacing the previous bespoke TLS proxy startup with alias registration at the app HTTPS port. PR #5 in ipfs-storage then adopts this new generic alias nginx config, and PR #4 refactors ipfs-storage further by replacing its TLS proxy server blocks with the standardised alias server blocks. This consolidation reduces duplication across services and ensures that any service consuming ssl-manager benefits from a consistent, centrally maintained nginx configuration pattern.
ssl-manager, ipfs-storage
Alias Proxy Stability and Crash Recovery Fixes
PRs #6, #7, and #11 in ssl-manager address reliability issues with the alias proxy component. PR #6 adds auto-restart behaviour on crash, while PR #7 strengthens this with a dedicated bash wrapper that handles crash recovery specifically for SIGSEGV signals, guarding against segmentation faults that auto-restart alone would not reliably catch. PR #11 fixes the alias proxy to correctly forward to the primary HTTPS endpoint, restoring full API access through the proxy layer. These fixes collectively harden the alias proxy against unplanned downtime.
ssl-manager
TLS Handshake Timeout to Prevent Thread Exhaustion
PR #8 in ssl-manager introduces a 10-second timeout on TLS handshakes, directly mitigating a thread exhaustion vector caused by health checks holding open connections indefinitely. Without this cap, long-lived or stalled handshakes from frequent health check polling could deplete the available thread pool and degrade service availability. The fix is a targeted, low-risk configuration change that improves the operational resilience of ssl-manager under sustained traffic and monitoring load.
ssl-manager
