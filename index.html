Full TcpStream parity across WIT, host, and SDKs
PRs #5 (wit), #746 (astrid), #42 (sdk-rust), and #3 (sdk-js) together deliver complete std::net::TcpStream surface mirroring for outbound TCP connectivity. The WIT interface gained a new net-connect-tcp world, the Astrid host runtime implemented matching host functions, and both the Rust and JavaScript SDKs expose the full API surface to capsule authors. This closes the gap between what capsule code can express and what the underlying Wasm host can execute for raw TCP networking. The work lands as a coordinated four-repo atomic feature, ensuring no SDK is left behind at the interface boundary.

SDK 0.6.1 dependency bump across all capsules
Fifteen capsule repositories each received a chore PR (e.g. capsule-skills #15, capsule-http #12, capsule-react #18) to bump astrid-sdk to version 0.6.1, which was cut in sdk-rust #40. This sweep ensures every first-party capsule is aligned with the new TcpStream surface and any other fixes included in the 0.6.1 patch release. Cargo-like publish/subscribe manifest schema migration
Fourteen capsules converted their manifests from an ad-hoc schema to a Cargo-inspired [publish]/[subscribe] layout with canonical WIT references, via PRs such as capsule-prompt-builder #13, capsule-memory #15, and capsule-context-engine #12. capsule-memory #15 additionally restored the on_before_prompt_build binding that had been dropped during an earlier refactor. The new schema makes inter-capsule message contracts explicit and machine-readable in a familiar format, improving tooling discoverability and reducing the chance of silent subscription mismatches. Canonical WIT refs in the manifest also allow the runtime to validate interface versions at install time rather than at first invocation.
capsule-prompt-builder, capsule-react, capsule-memory, capsule-registry, capsule-skills, capsule-shell, capsule-session, capsule-router, capsule-openai-compat, capsule-identity, capsule-http, capsule-hook-bridge, capsule-fs, capsule-context-engine
IPC publisher principal surface and Astrid 0.6.0 release
sdk-js #2 exposes the publisher principal field on IpcMessage, giving JavaScript capsule code the ability to inspect the identity of the sending capsule on every received message. astrid #744 tags the 0.6.0 runtime release that this SDK work tracks, and sdk-rust #40 cuts the 0.6.1 SDK patch on top of it. Surfacing the publisher principal is a prerequisite for capsules that need to enforce trust boundaries or route logic differently depending on message origin. Together these three PRs close the release loop and establish the baseline from which the 0.6.1 bump wave above proceeds.

IPC client configurable timeout for finality gadget
PR #11 introduces a configurable timeout on the IPC client used by the finality gadget, preventing indefinitely blocked calls when a peer is unresponsive. Without a timeout, a stalled IPC peer could hang the gadget's consensus participation loop, causing liveness degradation across the network. The change exposes the timeout as a tunable parameter so operators can balance responsiveness against transient network latency. This is a targeted reliability fix with no protocol-level changes, making it safe to deploy independently of any broader finality gadget version bump.
