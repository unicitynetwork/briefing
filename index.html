Connect Plugin: Session Persistence and Wallet Lock Handling
PRs #1 (unicity-claude-marketplace), #6 (sphere-sdk-connect-example), and #99/#101 (sphere-sdk) form a coordinated documentation and reliability push around the Connect plugin. The docs now cover session resume flows, LOCKED wallet state handling, and the safeSend utility, giving integrators a clear path for resilient session management. PR #99 complements this by fixing re-subscription to Nostr relay events after a reconnect, ensuring event delivery is restored automatically when connectivity drops. Together these changes harden the Connect integration surface and reduce the risk of silent failures in long-lived wallet sessions.

PostgreSQL Conflict Retry and JSON Validation Fixes
PR #12 in unicity-relay introduces automatic retry logic for serializable transactions that fail due to PostgreSQL serialization conflicts, a known hazard under concurrent write load with the SERIALIZABLE isolation level. This makes the relay more resilient under high-throughput conditions without requiring manual intervention or transaction redesign. PR #4 in unicity-node fixes invalid JSON handling, preventing malformed payloads from causing unhandled errors or silent data corruption at the node layer. Together these two fixes improve the reliability and correctness of the core network infrastructure.
unicity-relay, unicity-node
