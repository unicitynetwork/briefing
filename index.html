Sphere20 PRs merged
Fail-loud network config replaces silent defaults
PRs #467–#471 in sphere-sdk and #341–#342 in sphere deliver a three-phase campaign to eliminate silent testnet/mainnet network fallbacks. Phase 1 (#467) added regression guards specifically targeting the network-drop bug class. Phase 2 (#469) made the SDK throw immediately when network is absent rather than defaulting quietly. Phase 3 (#471) made NetworkConfig a required constructor argument and added a startup consistency assertion. The sphere repo was updated in lockstep (#341, #342) to pass network through Sphere.import, importFromLegacyFile, and Sphere.init, with SDK bumped to dev.4 then dev.5, and the integration test suite was fixed to use a shared TEST_NETWORK constant (#470).
sphere-sdk, sphere
Registry per-network cache fixes icon and token regressions
PR #466 in sphere-sdk threaded network identity through the registry init→create/load path and introduced a per-network cache so testnet2 icons and mint data are isolated correctly. PR #340 in sphere ensured network is passed to Sphere.init so the registry actually loads testnet2 assets, resolving an icons regression. PR #338 in sphere separately fixed token symbol resolution for fresh wallets, allowing symbols to appear without requiring a relogin cycle. Together these changes make the registry reliably network-aware from first launch.
sphere-sdk, sphere
Payments, swap, and profile correctness fixes
PR #453 split the payments local-commit and remote-publish phases to prevent race conditions between the two operations, and PR #451 corrected v2 send gating to check recipient chainPubkey rather than DirectAddress. PR #461 ensured terminal swaps are included in resolveSwapId and getSwaps and added write-side error codes, while #459 added fail-fast behaviour when transportPubkey is missing. PR #452 bounded the pre-shutdown publish-retry spin in the profile module to prevent indefinite loops, and #463 addressed HIGH-severity findings identified in the post-merge review of #453.
sphere-sdk
Transport, mux, escrow docs, and group chat housekeeping
PR #465 fixed an await gap in mux dispatch that could cause messages to be dropped during transport multiplexing. PR #460 added an assertion test confirming the mux subscription gate remains closed while modules are loading, preventing premature message delivery. PR #458 documented the escrow DIRECT address fallback for nametag resolution, and PR #462 captured findings from an investigation into single-coin faucet flakiness. In the sphere app, PR #339 disabled Group Chat by hiding the desktop icon and blocking access while the feature is not production-ready.
sphere-sdk, sphere

End-to-end h2c support across gateway and aggregator
PR #41 in aggregator-subscription added the ability to probe shard connectivity over HTTP/2 cleartext (h2c) when --upstream-h2c is set, fixing startup failures when upstreams are h2c-only. PR #42 extended the /health endpoint to be h2c-aware and extracted a shared AggregatorBlockHeightProbe to reduce duplication across health-check logic. On the infrastructure side, bft-ansible PR #8 bumped the testnet2 gateway image to commit f580750 as a prerequisite for the validator h2c fix, and PR #9 then cut over the gateway-to-HAProxy path on testnet2 to full h2c. The sequence ensures the entire testnet2 request path from gateway through aggregator operates correctly over h2c.
aggregator-subscription, bft-ansible
Java SDK gains networkId validation and cleaner mint params
PR #69 addresses issue #68 by adding a networkId field to the Java state-transition SDK and introducing verification logic to ensure transactions are rejected when the networkId does not match the expected network. The mint transaction parameter order was also reordered for consistency with other SDK conventions. This brings the Java SDK in line with the network-identity discipline being enforced across the broader Unicity stack and reduces the risk of cross-network transaction replay.
