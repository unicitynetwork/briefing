WebAssembly Component Model & WIT-Driven Type System Overhaul
PRs #649 and #10 together complete a foundational shift to the WebAssembly Component Model across the Astrid subsystem. PR #649 retargets the build to wasip2, bundles the SDK WIT definitions directly into the package, and normalises schema fields to snake_case for consistency. PR #10 migrates the capsule-hook-bridge to SDK 0.6.0, adopting wasmtime's Component Model runtime and replacing hand-written bindings with WIT-driven type generation. Together these changes align the build toolchain and runtime ABI, ensuring capsule hooks and the core Astrid build artefacts speak the same interface contract going forward.
