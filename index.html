Runtime E2E Coverage Gates Enforced in Core
PR #1070 introduces end-to-end coverage gates for the core runtime in the Astrid codebase. The change adds automated quality enforcement by requiring minimum coverage thresholds to pass before merges are accepted. This prevents regression in test coverage over time and gives the team a measurable signal on runtime reliability. Establishing these gates early in the pipeline reduces the risk of untested code paths reaching production environments.

Unicity Network5 PRs merged
Code-Driven Navbar and Footer Replace CMS Nav Models
PR #13 migrates the site's navigation structure away from CMS-managed nav models into hardcoded React components for both the navbar and footer. This removes a layer of CMS dependency from critical layout elements, reducing coupling and simplifying deployment. PR #12 complements this by enabling CMS-sourced images to render consistently across all pages and making the /blog header section inline-editable via the CMS. Together these two PRs from @KruGoL define a clearer boundary: CMS owns content and images, while code owns structure and navigation.
unicitynetwork/website-next
Production Deploy Defaults and IAM Observability Onboarding
PR #11 by @MastaP hardens the production deployment pipeline by defaulting the cms_tag and web_tag variables to the GHCR :latest image tag when no explicit value is provided, preventing broken deploys caused by empty tag inputs. PR #10 introduces a dedicated logs-reader IAM group for the website, along with onboarding documentation so new team members can gain read access to production logs without requiring elevated permissions. These infrastructure changes improve both deployment resilience and operational access control hygiene.
unicitynetwork/website-next
Benchmark Suite Expanded with Policies and Bug Fixes in Semanticd
PR #73 by @machmode significantly extends the benchmark test suite for semanticd, adding new datasets and benchmark policies tied to task #16 under ticket TK-260626. The expanded suite provides broader coverage of performance characteristics across varied data shapes, enabling more reliable regression detection for the semantic processing layer. Associated bug fixes included in this PR address issues surfaced during benchmark development, meaning the changes improve both test fidelity and runtime correctness simultaneously. This work lays the groundwork for data-driven performance targets as the semanticd component matures.
