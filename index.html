Sphere20 PRs merged
Fail-loud network config: no silent defaults
PRs #467, #469, and #471 in sphere-sdk delivered a three-phase overhaul that eliminates silent testnet/mainnet fallbacks when a network is not explicitly provided, replacing them with hard startup assertions that crash loudly on misconfiguration. Regression guard tests were added first in #467, the silent-default removal landed in #469, and #471 hardened NetworkConfig into a required parameter with a consistency assertion on startup. On the sphere side, #341 and #342 propagated these changes by threading the network argument through Sphere.import and importFromLegacyFile, then bumping the SDK to 0.8.0-dev.5. This eliminates an entire class of network-drop bugs where operations silently targeted the wrong chain.
sphere-sdk, sphere
Registry and icon loading fixed across networks
PR #466 in sphere-sdk fixed the registry initialisation path so that network is forwarded through init into create and load, and introduced a per-network cache so testnet2 icons and mint data are stored and retrieved independently of other networks. PR #340 in sphere applied the corresponding change so Sphere.init passes the network parameter, resolving an icons regression on testnet2. PR #338 in sphere fixed a related issue where fresh-wallet token symbols were not resolved without a relogin. Together these changes ensure the registry is correctly scoped per network from the very first wallet load.
sphere-sdk, sphere
Payments, swaps, and profile reliability fixes
PR #453 split the payments commit flow into separate local-commit and remote-publish phases, addressing sphere-sdk#444 and reducing the risk of partial state after a publish failure. PR #451 corrected the v2 send gate to check the recipient chainPubkey rather than a DirectAddress, and PR #461 included terminal swaps in resolveSwapId and getSwaps while adding write-side error codes (sphere-sdk#447). PR #459 added a fail-fast guard on missing transportPubkey in the swap flow (sphere-sdk#457), and PR #452 bounded the pre-shutdown publish-retry spin in the profile module to prevent indefinite loops (sphere-sdk#450). PR #463 addressed HIGH-severity findings from a post-merge review of #453.
sphere-sdk
Transport, mux gating, docs, and group chat
PR #465 fixed an await gap in the mux dispatch path of the transport layer (sphere-sdk#464), and PR #460 added a test asserting that the mux subscription gate remains closed for the full duration of module load (sphere-sdk#448). PR #458 documented the escrow DIRECT address fallback used during nametag resolution, and PR #462 published an investigation into single-coin faucet flakiness in the UXF suite. PR #339 in sphere disabled Group Chat by hiding the desktop icon and blocking access at the routing level.
