Sphere16 PRs merged
Detached Referral Visibility End-to-End
PRs #8 and #16 (sphere-api) extended the user-detail response to include the referral _id and expose detachedAt/detachedBy fields in referral entries. PR #2 (sphere-backoffice) fixed the _id fallback logic in the referral detach operation, and PR #5 (sphere-backoffice) renders a DETACHED badge on the corresponding admin rows. Together these changes give administrators a complete, accurate view of detached referral state directly in the back-office UI.
sphere-api, sphere-backoffice
OAuth Reconnect, Token Refresh, and Redis State
PRs #10, #11, and #12 (sphere-api) delivered a multi-part OAuth overhaul: Redis-backed pendingStates with a cacheRedis shim, an account-swap-safe Reconnect flow across all four social providers, and a refresh lock with Discord refresh-on-401 and rotation orphan cleanup. PR #13 (sphere-api) builds on this by setting failCode=ACCOUNT_NOT_CONNECTED when a social connection is absent, and PR #5 (sphere-quest-frontend) surfaces a Connect button to the user whenever that fail code is returned. PR #4 (sphere-quest-frontend) adds an account-mismatch toast so users are immediately informed when a social account conflict is detected during the OAuth flow.
sphere-api, sphere-quest-frontend
SSRF Mitigation and External API Hardening
PR #15 (sphere-api) introduced validation of external-API URLs to block server-side request forgery vectors, and gated the /test endpoint so it is only accessible on published projects. This closes a class of SSRF vulnerabilities where an attacker could supply arbitrary URLs to internal infrastructure. The change is a targeted security fix with no user-facing functional impact but significantly narrows the attack surface of the verifier subsystem.
sphere-api
Admin Codebase Cleanup and Infrastructure Hygiene
PR #14 (sphere-api) applied post-umbrella final-review follow-ups, and PR #9 (sphere-api) dropped the dead actorRole='user' arm from the referrals logic. PR #4 (sphere-backoffice) renamed UserDetailModal to UserDetail, removing dead modal scaffolding, while PR #3 (sphere-backoffice) introduced component test infrastructure using React Testing Library and happy-dom. PR #2 (sphere-apps) renamed 'Boxy Run' to 'Boxy Run Arena' and updated its URL, keeping the app registry consistent with the current product naming.
sphere-api, sphere-backoffice, sphere-apps
