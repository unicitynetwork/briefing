Sphere23 PRs merged
Intent validation hardening and payload budgeting
A coordinated chain of PRs tightened the intent pipeline end-to-end. wallet-api PR #103 introduces an env-tunable 64 KiB per-intent payload cap alongside a per-owner lifetime intent byte budget, giving operators levers to prevent abuse. sphere-sdk PR #671 makes deterministic putIntent validation rejections terminal rather than retriable, and isolates resync replay paths to avoid compounding failures. sphere PRs #427 and #422 bumped sphere-sdk through 0.11.9 and 0.11.10 to surface these protections in the main app, and a flaky pollOrder listener-leak test caused by real-timer races on loaded CI runners was corrected in sphere PR #429.
wallet-api, sphere-sdk, sphere
Payment sync UX and wallet-api reliability fixes
sphere-sdk PR #669 introduced a new SEND_SYNC_PENDING tag for post-commit mirror-sync failures, replacing a silent error state with a reassuring pending-success signal. sphere PR #426 consumes this by presenting SEND_SYNC_PENDING to users as pending-success, improving perceived reliability. On the reliability side, sphere-sdk PRs #666 and #667 fixed auth challenge validation to use server timestamps instead of the device clock, and treated lineage CONFLICT responses during token-storage save as benign no-ops rather than hard errors. sphere-sdk PR #668 added retry logic on POST /v1/inventory/apply for transient network failures, safe because the endpoint is idempotent by transferId, a contract now formally documented in wallet-api PR #101. wallet-api PR #99 also patched a §8.2 step 6 validation fallback to bare PaymentAssetCollection, and sphere PR #421 rolled up the sphere-sdk 0.11.7 bump with wallet-api timeout and retry config.
sphere-sdk, sphere, wallet-api
Marketplace quest tracks and backoffice admin UX
sphere PR #424 and sphere-api PR #51 together deliver quest tracks: a new endpoint exposes track data server-side while the UI gains quest track display and project-page polish. sphere-backoffice PR #22 and sphere-api PR #50 add server-side admin search and hide exhausted quests from listings, alongside a rejection history panel with reason presets for moderators. sphere-ui PR #17 polishes the logo tile component, removing unwanted cropping and gloss overlays on real logos and switching chips to a neutral style. sphere-dev-portal PR #24 clarifies in the quest creation UI that Max Completions is a global cap, not a per-user limit, reducing operator confusion. Dependency bumps in sphere-dev-portal PR #25, sphere-backoffice PR #23, and sphere-quest-frontend PR #12 aligned all downstream apps to sphere-ui 0.1.30 and sphere-sdk 0.11.8.
sphere, sphere-api, sphere-ui, sphere-backoffice, sphere-dev-portal, sphere-quest-frontend
