Astrid10 PRs merged
Load-time capsule introspection replaces static tool baking
PRs #979, #984, #988, and #990 together deliver and then supersede a build-time tool-baking pipeline. #979 first introduced baking tool descriptors into meta.json at build time, and #984 added a flag marking whether a capsule's tool surface was captured during that build. #988 then pivoted the architecture to describe un-baked capsules at load time instead, making the static tool surface automatic without requiring a dedicated build step. #990 completed the transition by removing the build-time tools.json mechanism entirely now that load-time describe supersedes it. The net result is a simpler, more maintainable capsule lifecycle with no manual tool-surface bookkeeping.
unicity-astrid, unicity-sage
Sage MCP tools/list served from kernel metadata
PRs #17 and #19 in the sage repo overhaul how the MCP server exposes the tool catalogue. #981 first extended the kernel to carry per-capsule opaque metadata in the capsules_loaded payload, giving downstream consumers a richer data source. #19 then wired sage-mcp to serve tools/list directly from that static kernel metadata, retiring the previous fan-out approach that queried each capsule individually. #17 added cache eviction so the tool cache is invalidated whenever the active capsule set changes, preventing stale tool listings after installs or removals. Together these changes make tool discovery faster, more consistent, and decoupled from per-capsule round-trips.
unicity-sage, unicity-astrid
Live capsule hot-swap and unload without restart
PRs #969 and #971 bring dynamic capsule lifecycle management to the kernel. #969 introduced hot-swap support via the reinstall/update path, allowing an already-loaded capsule to be upgraded in place without restarting the kernel process. #971 added the complementary KernelRequest::UnloadCapsule variant, enabling live unload of a capsule when it is removed. These two capabilities together mean operators can install, upgrade, and remove capsules in a running system, significantly improving operational flexibility and reducing downtime during updates.
unicity-astrid
Graceful 'unknown hook action' denial treated as no-tools
PR #986 tightened error handling in the capsule build-capture path. Previously, a deny response carrying an 'unknown hook action' reason could propagate ambiguously through the tool-surface capture pipeline. The fix treats that specific denial as a no-tools signal rather than an error, allowing the build process to continue cleanly when a capsule simply does not implement a given hook. This prevents spurious build failures and aligns deny semantics with the broader load-time introspection model introduced in the surrounding PRs.
unicity-astrid

Redis, Postgres, and shutdown timeouts bound for resilience
PR #73 addresses issue #72 by introducing explicit upper-bound timeouts on Redis connections, Postgres queries, and graceful shutdown sequences in the wallet API. Previously, slow or unresponsive dependencies could cause requests to hang indefinitely, exhausting worker threads and degrading the entire service. The fix adds fail-fast logic that returns HTTP 503 to callers as soon as a timeout is breached, making failures visible and bounded rather than silent and unbounded. This improves overall system resilience and makes the API's behaviour under dependency failures predictable and operator-observable.
