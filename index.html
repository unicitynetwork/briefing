Rust SDK packaging and serialization fixes
Three PRs from @joshuajbouw landed across the weekend to stabilize the Rust SDK ahead of and following the 0.6.0 release cut in PR #34. PR #36 resolves a cargo publish failure by embedding WIT files directly inside the crate and flattening the wit/ directory layout, ensuring downstream consumers receive the interface definitions without manual steps. Together these fixes make the 0.6.0 crate publication reliable and the serialization contract consistent for any SDK integrators.

Unicity Network1 PRs merged
CBOR type tag registry for unicity-ids
PR #5 from @ahtotruu introduces a formal registry of CBOR type tags into the unicity-ids repository. CBOR type tags provide a standardized mechanism for annotating serialized data with semantic type information, and a centralized registry ensures consistent tag assignment across all components that encode or decode identity-related structures. This prevents tag collisions between independently developed modules and makes the encoding scheme self-describing and auditable. Establishing this registry now is a foundational step for any cross-component or cross-language interoperability that relies on CBOR-encoded identity payloads.
