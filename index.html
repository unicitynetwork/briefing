Astrid34 PRs merged
Streaming HTTP airlock added to capsule runtime
astrid #509 introduces a handle-based streaming HTTP airlock with three new host functions (astrid_http_stream_start, astrid_http_stream_read, astrid_http_stream_close), letting capsules consume HTTP responses chunk-by-chunk instead of buffering entire bodies. sdk-rust #4 adds the companion SDK surface — HttpStreamHandle, stream_start, and stream_read — so capsule authors can use the feature from Rust without touching raw host calls. sdk-rust #5 cuts release 0.2.2 to ship this API. This unblocks real-time streaming use cases such as LLM token streaming inside capsules.
astrid, sdk-rust
Daemon extracted; CLI becomes pure socket client
astrid #507 splits the monolithic binary into astrid-daemon (kernel daemon) and astrid-build (capsule build tooling), leaving the CLI as a pure socket client with no embedded runtime. Three new lifecycle commands — start, status, stop — let operators manage the daemon process explicitly. astrid #513 follows up with end-to-end boot pipeline fixes, a headless mode, and post-install onboarding flow, closing issues #516 and #517. astrid #512 bumps the entire workspace to 0.4.0 to ship these changes.
astrid
Tool descriptions and SDK 0.3.0 shipped across all capsules
sdk-rust #7 extends the #[capsule] proc macro so that Rust doc comments (///) on #[astrid::tool] methods become tool descriptions in the exported JSON schema, and adds an inline mutable flag. sdk-rust #8 releases this as 0.3.0 with a minor version bump because the schema export format changed from a plain tools map to {tools, description}. Six capsules (openai-compat, identity, skills, shell, http, fs) each landed a feat PR adding doc comments to their tools, followed by SDK bump PRs across all 13 capsules in the org. LLMs calling these tools now receive structured descriptions to guide tool selection.
sdk-rust, capsule-openai-compat, capsule-identity, capsule-skills, capsule-shell, capsule-http, capsule-fs
Capsule registry deduplication and dead IPC subscription removed
capsule-skills #4 removes duplicate [[tool]] entries from Capsule.toml that were causing each skill tool to be registered twice in the kernel's tool registry, producing phantom duplicates visible in astrid capsule list-tools. capsule-cli #3 removes the agent.v1.stream.delta IPC subscription from the CLI proxy capsule; this event bus topic no longer exists, so the subscription was dead overhead on every CLI invocation. capsule-openai-compat #1 adds placeholder hints to all [env] fields in Capsule.toml and updates the default model from gpt-4o to gpt-5.4.
capsule-skills, capsule-cli

Wallet events, onchain achievements, and identity-aware connect
sphere #284 adds dApp notifications on wallet switch and logout events, and introduces a video background poster for the UI. sphere-quest #6 implements onchain achievement sync, enabling quest completions to be recorded and verified on-chain. sphere-sdk #92 introduces an identity-aware connect protocol, so dApps connecting through the SDK can incorporate identity context directly in the handshake. Unicity Network3 PRs merged
Receipt removed from certification request/response pipeline
state-transition-sdk #101 and aggregator-go #128 coordinate to strip the receipt field from both the certification request and response structures, tracking issue #92. This simplifies the certification protocol by removing an artifact that is no longer needed at the request/response boundary. The change touches both the Go aggregator and the SDK in tandem, so consumers of either must update to the new schema. unicity-relay #4 lands a Metrics implementation alongside these changes, adding observability instrumentation to the relay layer.
state-transition-sdk, aggregator-go
