Purge stale WASI target refs across all capsules
PRs #7, #18, #21, #14, #14, #16, #16, #17 swept through every capsule repo to remove obsolete wasi-target references from release workflows. These stale references were leftovers from an earlier build target strategy and risked producing broken CI pipelines or incorrect release artifacts. The cleanup is purely chore-level with no functional changes to capsule behaviour, but it hardens the release automation going forward.
capsule-agents, capsule-memory, capsule-identity, capsule-http, capsule-hook-bridge, capsule-fs, capsule-context-engine, capsule-cli
Migrate all capsules to per-domain WIT + wasm32-unknown-unknown
A coordinated breaking-change wave (PRs #16 through #6 across 17 repos) replaced the shared monolithic WIT interface with per-domain WIT definitions and switched the compile target from wasm32-wasi to wasm32-unknown-unknown. Per-domain WIT gives each capsule a tightly scoped interface surface, reducing cross-capsule coupling and enabling independent versioning of component contracts. The target switch to wasm32-unknown-unknown removes the WASI runtime dependency, making capsules portable to any compliant Wasm host without requiring a WASI preview layer. This is a semver-breaking change for all downstream consumers who embed or link these capsules.
capsule-skills, capsule-shell, capsule-session, capsule-router, capsule-registry, capsule-react, capsule-prompt-builder, capsule-openai-compat, capsule-openai, capsule-memory, capsule-identity, capsule-http, capsule-hook-bridge, capsule-fs, capsule-context-engine, capsule-cli, capsule-agents
Refresh Homebrew tap SHAs after 0.7.0 release retag
PR #6 updated the SHA256 checksums in the Homebrew formula following a retag of the 0.7.0 release. A retag invalidates previously computed archive hashes, so without this fix Homebrew installations would fail integrity verification. The change is purely metadata and does not alter the distributed binary content.
homebrew-tap

Profile mode token migration and safe re-enablement
PRs sphere#307 and sphere-sdk#289 deliver the complete legacy-to-Profile token migration path that was missing when sphere#305 was reverted via sphere#306. Sphere-sdk#289 implements bidirectional legacy↔Profile token storage migration so that existing tokens are transparently carried over when the Profile-backed providers are activated. Sphere#307 then re-enables Profile mode in the UXF build now that the safe migration path exists, making the earlier revert in #306 temporary as intended. Together these PRs close the data-loss risk that caused the original rollback.
sphere, sphere-sdk
OpLog oversize writes fixed with CID-reference indirection
PRs #287 and #288 address the same root issue (tracked in #285) across CommunicationsModule and GroupChatModule: writes that exceeded OpLog size limits were being stored inline, causing failures under large payloads. Both PRs migrate oversized entries to CID-references, storing the full payload in the block store and recording only the content identifier in the OpLog. PR #288 supersedes or complements #287, suggesting an iterative fix cycle within the same bug thread. This prevents silent data truncation and keeps OpLog entries within protocol-mandated bounds.
sphere-sdk
Profile transport reliability and blockstore performance
A series of targeted fixes hardened the Profile storage and transport stack: PR #273 decoupled per-flush HEAD verification from the at-least-once delivery gate to prevent spurious re-sends; PR #279 bounded helia.blockstore.get calls with an LRU cache and in-flight deduplication, capping memory growth under concurrent block fetches; PR #281 added a CBOR fallback for SENT-record deserialisation and introduced an aggregator-spent recovery sweep; and PR #283 resolved two §D residuals covering deterministic asset ordering and invoice round-trip correctness. Collectively these changes improve resilience for long-running Profile sessions with large block graphs.
sphere-sdk
Observability, dedup persistence, and payments hardening
PR #276 introduced toggleable verbose debug logging with namespace filters, span tracking, and field redaction, giving operators fine-grained visibility without leaking sensitive data in production. PR #277 persisted Nostr event deduplication state across process restarts, closing a gap where duplicate events could be reprocessed after a crash or restart. PR #271 classified VerificationError(Recipient address mismatch) as a permanent payment failure, stopping infinite retry loops for structurally invalid invoices. Sphere-cli PR #29 fixed a UX regression from sphere-sdk#282 by routing wallet use confirmation output to STDERR, keeping STDOUT clean for scripted consumers.
