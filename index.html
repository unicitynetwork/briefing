Moderation pipeline: submission gates and rejection flow
A multi-PR effort hardened the project submission and moderation lifecycle across the stack. PR #38 (sphere-api) enforces that projects must supply an App URL and agree to submission criteria before entering review, while PR #39 persists rejection reasons and clears them automatically on resubmit. On the backoffice side, PR #15 surfaces the App URL in the moderation queue and adds a one-click Reject action on the project overview, and PR #17 (backoffice) adds anti-abuse context including reject counts, org stats, and a Ban Organisation action to give moderators richer signal. A submission criteria modal and App URL gate were also added to the dev portal in PR #17 (dev-portal), closing the loop between what is required and what moderators can see.

Unicity Network11 PRs merged
UNIP-01 single-owner identity bindings across SDK and relay
PR #25 in unicity-tokens-relay implements the core UNIP-01 protocol change: single-owner identity bindings that tie a nametag to exactly one controlling key. PRs #9 (nostr-js-sdk) and #6 (nostr-sdk, Java mirror) implement the corresponding client-side ownership marker and marker-preferring resolution logic, ensuring clients can deterministically resolve identities when multiple candidate records exist. PR #10 in nostr-js-sdk adds a live single-owner end-to-end test against a real UNIP-01 relay, validating the full stack integration rather than relying on mocks alone. Together these PRs represent the complete cross-layer delivery of the UNIP-01 identity ownership model.
nostr-js-sdk, nostr-sdk, unicity-tokens-relay
Testnet2 gateway ECS migration and ALB cutover
PR #13 in bft-terraform cuts the ALB port-443 default action over to the ECS Fargate service, completing the traffic migration for the testnet2 subscription gateway. PR #14 adds Cloud Map ECS service discovery so that Alloy can scrape /metrics endpoints from dynamically scheduled Fargate tasks without manual target configuration. PR #25 in bft-ansible adds a complementary Alloy discovery.dns scrape block that resolves ECS Fargate gateway tasks via DNS, completing the observability plumbing for the new deployment model.
bft-terraform, bft-ansible
Testnet2 Alloy monitoring fixes and dashboard consolidation
PR #23 fixes the Alloy healthcheck by correcting the bash /dev/tcp probe syntax and updates the CloudWatch namespace away from a deprecated alias, resolving silent monitoring failures. PR #24 follows up by dropping a shell variable that docker compose could inadvertently blank, making the healthcheck robust across different invocation contexts. PR #22 consolidates ECS autoscaling metrics into the existing gateway dashboard, producing a single unified 'V2 — Subscription Gateway' view and reducing dashboard sprawl.
bft-ansible
Benchmark policy relocation and workflow documentation
PR #74 (TK-260629) relocates benchmark policies out of the server load directory in semanticd, separating concerns between production-loaded policies and benchmarking artefacts. The accompanying documentation clarifies the intended workflow for running and maintaining benchmarks, reducing the risk of benchmark policies accidentally influencing server behaviour. This is a low-risk housekeeping change that improves long-term maintainability of the semanticd test and performance infrastructure.
