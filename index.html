Astrid21 PRs merged
HTTP 1.1.0: Per-Request Control and Streaming Uploads
PRs #17 (wit), #1050, #1054 (astrid), and #60 (sdk-rust) collectively land the astrid:http@1.1.0 interface across the full stack. The WIT contract (#17) introduces per-request control surfaces, streaming upload support, and trailer handling. The host implementation (#1050) registers and wires up the new interface inside the Astrid runtime, while #1054 adds operator-configurable [http] host limits alongside follow-up fixes. The Rust SDK (#60) exposes the new control surface to capsule authors, completing the end-to-end delivery of the upgraded HTTP capability.
wit, astrid, sdk-rust
Conversation Threads: Management, Gateway, and Session CLI
PRs #16 (wit), #1045 (astrid), and #17 (capsule-session) introduce a full conversation-management capability across the platform. The astrid-bus:session 1.1.0 WIT contract (#16) formalises conversation metadata, verbs, and agent-tool hooks at the interface level. The gateway implementation (#1045) exposes REST endpoints for listing, fetching, managing, searching, and live-feeding conversation threads over HTTP. capsule-session (#17) backs these with metadata storage, verb dispatch, and CLI tooling, giving operators and agents a complete handle on conversation lifecycle.
wit, astrid, capsule-session
Elicitation, Local-Egress Consent, and Agentic Gateway
PRs #1029, #1030 (astrid), and #34 (capsule-cli) build out the operator-consent and agent-elicitation surface. #1029 introduces transport-origin markers, operator-consent gating for local-egress, and a guided pre-bless flow so capsules cannot silently reach local network resources. #1030 adds POST /api/agent/elicit-response to the gateway and enforces same-principal validation so elicit replies cannot be injected across principals. capsule-cli #34 fixes outbound IPC demultiplexing to key on both session and principal, closing a session-isolation gap that would have undermined the new consent model.
astrid, capsule-cli
Core Newtypes, IPC Topics, and Distro Seeding
A series of refactoring and infrastructure PRs tighten type safety and operational coverage across the codebase. #1048 and #1057 introduce device-key identifier newtypes and round out device public-key traits, eliminating stringly-typed key handling in the core. #1038 wraps IPC topic strings in a Topic newtype with typed constructors, preventing topic-string misuse at compile time. #1044 and #1034 newtype group-config and principal-profile fields respectively, continuing the systematic hardening of config boundaries. #1040 fixes a kernel bug where rate-limited management errors were not published on the response topic, and #1017 delivers headless and offline distro seeding (tracking issue #964), enabling deployment in air-gapped environments.
astrid

Browser Compatibility Fixes: AbortSignal and crypto.randomUUID
PRs #618 and #620 (sphere-sdk) patch two Web API compatibility gaps that caused failures in older browsers and WebViews. #618 guards AbortSignal.timeout behind a feature-check since the method is absent in older environments, while #620 routes bare crypto.randomUUID() calls through a browser-safe polyfill helper to handle contexts where the Crypto API is restricted. These fixes were then consumed by sphere via back-to-back SDK bumps in #387 (0.10.3→0.10.4) and #390 (0.10.4→0.10.5), ensuring the production wallet UI picks up both patches without requiring application-level changes.
sphere-sdk, sphere
Wallet Metrics Gauges and Immutable Asset History
PRs #76 and #78 extend the wallet-api with observability and data-integrity improvements. #76 introduces Prometheus-style product gauges tracking wallet count, completed transfers, and active token volume (#75), giving operators real-time visibility into platform health. #78 makes the created_at field on inventory_items immutable as a fix-forward measure (#77), ensuring asset creation timestamps cannot be overwritten and preserving a reliable audit trail for asset history.
wallet-api
