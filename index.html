Astrid-owned I/O layer replaces WASI standard
PR #8 swaps out the generic wasi:io dependency for a purpose-built astrid:io@1.0.0 interface covering error, poll, and streams primitives. This gives Astrid direct ownership over its I/O contracts rather than inheriting upstream WASI semantics. The change is a foundational prerequisite for any Astrid-specific I/O behaviour that diverges from the WASI spec.

Profile IPFS pinning: sidecar, CAR blocks, fallback
PRs #259 and #260 wire up the IPFS instant-pin sidecar as the primary pinning path, with /api/v0/dag/put as a fallback for profile data. PR #237 pins CAR blocks directly to the local Helia blockstore as the primary store, and PR #212 reverted an earlier monolithic raw-codec approach before PR #214 introduced hierarchical UXF bundle pinning via Option C. On the infrastructure side, ipfs-storage PRs #7 and #9 add the sidecar write-through cache and the /api/v0/dag/put nginx location to support CAR uploads end-to-end.
sphere-sdk, unicitynetwork/ipfs-storage
RFC-251 pointer coordination: win-broadcast across devices
PR #254 introduced RFC-251 documenting same-identity cross-device pointer coordination, and PR #258 records the RFC flipping from Approach A to Approach D. PR #257 (draft) implements Phase 1 of the win-broadcast strategy under Approach D, establishing which device wins the pointer-publish race. PR #261 ensures sphere.destroy() flushes pending pointer publishes before exit so no in-flight updates are lost on shutdown. Together these PRs lay the groundwork for deterministic multi-device pointer convergence.
sphere-sdk
Payment and oracle bug fixes: recovery, finalize, isSpent
PR #256 fixes HD-index recovery and adds a diagnostic for finalize mismatches in the payments layer. PR #252 unwraps the OracleProvider wrapper that was preventing V6-RECOVER finalize from completing correctly, and PR #233 classifies the resulting InvalidJsonStructureError as a permanent failure to stop futile retries. PR #244 switches isSpent to the canonical get_inclusion_proof call, with PR #246 applying follow-up corrections to isSpent, pointer-publish, and OrbitDB connect behaviour surfaced after #243.

Profile sync stability: walkback, OpLog, flake elimination
PRs #248 and #249 address WALKBACK_FLOOR coalescing, pre-Mux buffering, and downward localVersion reconciliation for same-identity walkback scenarios. PR #250 migrates adapter writes to the OpLog envelope, and PR #242 decouples the at-least-once gate from aggregator publish while extending walkback retry logic. On the CLI side, sphere-cli PR #28 refuses to start when a sphere daemon holds the OrbitDB lock, and PRs #216, #220, and #221 eliminate a series of deterministic and stress-surfaced test flakes across profile and pointer-monotonicity suites.

Nginx cache correctness and sidecar fast-path fixes
PR #8 reverts the /ipfs and /api/v0/block/get sidecar fast-paths introduced in #6 after they caused correctness issues. PR #10 partitions the proxy_cache_key by both the ?format query parameter and the Accept header, preventing cache collisions between different response formats. These two fixes restore reliable content negotiation for cached IPFS responses before the sidecar feature is re-introduced cleanly.
ipfs-storage
Infra probe v0.5.0: e2e faucet and aggregator checks
PR #3 ships infra-probe v0.5.0 with end-to-end faucet verification, local network testing support, and critical-path aggregator health checks. These additions give operators automated signal on whether the faucet and aggregator are functioning correctly in a live or local environment. The release moves the probe from basic liveness checks toward meaningful functional coverage of the network stack.
