Astrid2 PRs merged
WebAssembly Component Model & SDK 0.6.0 Migration
PRs #649 and #10 together advance the Astrid subsystem to a fully Component Model-aligned WebAssembly stack. PR #649 retargets the build to wasip2, bundles the SDK WIT definitions directly into the crate, and normalises schema field naming to snake_case for consistency across generated types. PR #10 migrates the capsule-hook-bridge to SDK 0.6.0, adopting wasmtime's Component Model runtime and switching to WIT-driven type generation throughout. Together these changes ensure that capsule hooks and the core Astrid build surface share a coherent, standards-compliant interface, reducing the risk of ABI mismatches as the component ecosystem matures.
