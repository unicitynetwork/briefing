WebAssembly Component Model & SDK 0.6.0 Upgrade
PRs #649 and #10 together advance the Astrid stack to full WebAssembly Component Model compliance. PR #649 retargets the build to wasip2, bundles the SDK WIT definitions directly into the repository, and normalises schema fields to snake_case for consistency across generated bindings. PR #10 migrates the capsule-hook-bridge to SDK 0.6.0, adopting wasmtime's Component Model runtime and replacing ad-hoc type definitions with WIT-driven types throughout. Together these changes establish a single, contract-first interface boundary between host and guest components, reducing manual FFI glue and making future SDK upgrades easier to track.
