Astrid7 PRs merged
IPC publish/subscribe ACL with subtree wildcard matching
PR #883 introduces trailing `*` wildcard support in the IPC pub/sub ACL layer, making a star suffix match the entire subtree of a topic namespace rather than a literal string. PR #25 in capsule-cli follows up by correcting the IPC ACL configuration: it drops crashing admin-response subscribes and explicitly permits MCP front-door subscriptions. Together these changes make the ACL rules expressive enough for real deployment topologies while eliminating a class of crash-on-subscribe bugs.
astrid, capsule-cli
Guest error surfacing and principal profile resolution fixes
PR #885 fixes a silent run-loop crash in the host daemon by surfacing ERROR-level logs emitted inside the guest up to the daemon log, making previously invisible failures observable. PR #878 resolves the invoking principal's profile on the `ipc::recv` path, ensuring callers are correctly identified during inbound IPC dispatch. PR #56 in sdk-rust ensures the `tool_describe` macro publishes its descriptor into the describe fan-out, which was previously missing and would cause tool discovery to silently omit entries. Collectively these patches close several hard-to-diagnose silent-failure modes in the guest/host boundary.
astrid, sdk-rust
Astrid 0.8.0 release and CI housekeeping
PR #854 cuts the 0.8.0 release of Unicity Astrid, consolidating all recent IPC, host, and capsule fixes into a versioned artifact. PR #889 generalises the PR template by renaming the 'Test Plan' section to 'Verification', reflecting a broader definition of correctness evidence that includes integration and manual checks. These are low-code but high-process changes that stabilise the release baseline and improve contributor workflow consistency.

V2 engine cutover: remove v1 SDK, mint UnicityIdToken
PR #480 performs the headline breaking change: the v1 state-transition-sdk is fully removed, completing the cutover to the v2 engine. PR #481 mints and stores the v2 UnicityIdToken at nametag registration time, tying identity tokens to the new engine's lifecycle. PR #345 and its merge PR #346 land the sdk-migration-v2 feature branch into the main sphere app, while PR #344 bumps sphere-sdk to 0.8.0-dev.6 to pick up per-network storage isolation. PR #484 refreshes all SDK documentation to reflect the v2 engine surface, ensuring the reference material is no longer describing removed or renamed APIs.
sphere-sdk, sphere
Per-network storage isolation and payment verification
PR #472 delivers per-network storage isolation for the v2 finish, scoping the token DB, KV store, and IPNS records by a `chainPubkey` identifier so data from different networks cannot collide. PR #485 network-scopes the `PENDING_V2_DELIVERIES` journal specifically, closing a gap where pending delivery state was shared across networks. PR #479 adds verification of incoming v2 transfers and makes `validate()` engine-aware, preventing a node from accepting transfers that belong to the wrong engine version. PR #343 disables NIP-29 group chat at SDK init in the sphere wallet, removing an unintended dependency on that Nostr extension.
sphere-sdk, sphere
Trader CLI, swap escrow, and local token minting
PR #49 adds `sphere trader spawn/stop` wrapper commands and `sphere host local-*` primitives to the CLI, giving operators a direct handle on trader lifecycle and local host operations. PR #475 delivers a roundtrip soak test, a demo playbook, a spec drift audit, and an investigation into issue #473, hardening the trader path end-to-end. PR #468 hardcodes and exports `DEFAULT_ESCROW_ADDRESS=@escrow-test-02` in the swap module, providing a stable test escrow reference for development. PR #45 replaces the HTTP faucet dependency with local test-token minting, making CLI test flows runnable without external network access. PR #483 fixes all price fields to use decimal-string bigints, preventing numeric precision loss in market data serialisation.
sphere-cli, sphere-sdk
IAM deploy policy documentation for GHA
PR #6 documents per-project GitHub Actions deploy policies in the sphere-infra IAM layer. This establishes an auditable record of which GHA workflows hold which deployment permissions across sphere sub-projects. Having this codified in the infra repo reduces the risk of permission sprawl as the number of automated release pipelines grows.

Training refactor: shared REPO_ROOT and portable model writes
PR #57 in the sif repository refactors the training pipeline to share a single `REPO_ROOT` reference across scripts, eliminating duplicated path definitions that could diverge. The PR enforces portable `base_model` write paths, ensuring checkpoints and model artefacts land in locations that are valid regardless of the execution environment. These changes improve reproducibility and reduce the likelihood of silent misconfiguration when running training jobs in different CI or cloud contexts.
