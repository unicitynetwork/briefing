Connect Plugin: Session Persistence and Lock Handling
PRs #1 (unicity-claude-marketplace), #6 (sphere-sdk-connect-example), and #101 (sphere-sdk) form a coordinated documentation push across the Connect plugin ecosystem by @KruGoL. The updates cover session resume flows, LOCKED wallet state handling, and the safeSend utility to gracefully manage sends when a wallet is in a locked state. Host-side requirements are also documented in the example repo, giving integrators a complete reference for robust session lifecycle management. Together these changes significantly lower the integration barrier for developers building on the Connect plugin.
unicity-claude-marketplace, sphere-sdk-connect-example, sphere-sdk
Nostr DM Reliability and Relay Reconnection Fixes
@MastaP addressed two related Nostr reliability issues across sphere and sphere-sdk. PR #289 (sphere) fixes bugs in Nostr direct message handling, improving message delivery correctness. PR #99 (sphere-sdk) resolves a gap where event subscriptions to Nostr relays were not re-established after a reconnect, preventing missed events in long-running sessions. Combined, these fixes make the Nostr messaging layer significantly more resilient to connection interruptions.
sphere, sphere-sdk
App Reload Stability Fix

The fix ensures the app recovers cleanly from a reload event without requiring manual user intervention. This improves general UX reliability, particularly for users who frequently background or restart the app.

PostgreSQL Conflict Retry and JSON Validity Fixes
PR #12 in unicity-relay by @MastaP introduces automatic retry logic for serializable transactions that fail due to PostgreSQL serialization conflicts, a common occurrence under concurrent write load with SERIALIZABLE isolation. This makes the relay more resilient to high-concurrency scenarios without requiring manual intervention or losing transactions. PR #4 in unicity-node by @lploom fixes invalid JSON being produced or consumed by the node, which could cause parse failures downstream in any service relying on that output. Together these two fixes improve the reliability and correctness of the core network infrastructure layer.
unicity-relay, unicity-node
